# MAGI 事務所營運工作台 Roadmap

本文件定義 MAGI / Paperclip 事務所營運工作台的十項完工範圍。計時功能不列入本期。

## 完工定義

- 十項功能均有可用 API 或介面入口。
- 重要資料不可混案、混案由或污染學習紀錄。
- 對外功能預設採安全模式；公開上傳入口、完整瀏覽稽核預設關閉。
- 每輪改動需通過單元測試、CI smoke，以及可在本機 live smoke 驗證。

## Phase 1：安全底座

1. 修正學習中心  
   統一呈現人工改正紀錄；跨案套用必須符合案由隔離規則。

2. 品質 / 幻覺審核層  
   檢查內部案號外洩、模型思考標記、待確認欄位、引用來源不足與案號不一致。

3. 輕量權限與高風險操作紀錄  
   完整開啟/瀏覽稽核預設關閉，只顯示刪除、搬移、分享、匯出、歸檔、學習回饋等高風險事件。

## Phase 2：每日工作流

4. 期限與風險儀表板  
   整合待辦、行事曆、法扶長期未結與逾期風險，依嚴重度排序。

5. 對外文件產生包  
   依案件/案由產出可複製的當事人待補資料清單與說明文字。

6. 文件證據索引 / 事實時間線  
   從 `document_index` 建立案件文件時間線，供書狀與附件清單引用。

## Phase 3：收案與當事人互動

7. 收案 / 諮詢漏斗  
   記錄潛在案件、諮詢摘要、聯絡方式與下一步，並同步跑利益衝突檢查。

8. 利益衝突檢查  
   查詢既有案件、當事人與對造資料；對造候選視為高風險，當事人候選需人工確認。

9. 客戶 / 當事人入口  
   本期採資料包模式，先提供可複製的安全入口內容；真正公開上傳入口保留設計但預設不開啟。

## Phase 4：管理視角

10. 營運報表  
    不含計時，統計進行中案件、已結案、待辦、逾期待辦、文件索引、實務見解、法扶案件、學習與收案事件。

## 目前落地入口

- 前端：Paperclip 側欄 `事務所營運`
- API：
  - `GET /api/osc/saas/overview`
  - `POST /api/osc/saas/conflict-check`
  - `POST /api/osc/saas/intake`
  - `POST /api/osc/saas/quality-check`
  - `POST /api/osc/saas/client-packet`
  - `GET /api/osc/saas/timeline`

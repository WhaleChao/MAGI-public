# MAGI 管理工具 Roadmap

本文件定義 MAGI / Paperclip 管理工具的十項完工範圍。計時功能不列入本期。

## 完工定義

- 十項功能均有可用 API 或介面入口。
- 重要資料不可混案、混案由或污染學習紀錄。
- 對外功能預設採安全模式；公開上傳入口、完整瀏覽稽核預設關閉。
- 每輪改動需通過單元測試、CI smoke，以及可在本機 live smoke 驗證。

## Phase 1：安全底座

1. 修正學習中心  
   統一呈現人工改正紀錄；跨案套用必須符合案由隔離規則。

2. 品質 / 幻覺審核層  
   檢查內部案號外洩、模型思考標記、待確認欄位、引用來源不足與案號不一致。

3. 輕量權限與高風險操作紀錄  
   完整開啟/瀏覽稽核預設關閉，只顯示刪除、搬移、分享、匯出、歸檔、學習回饋等高風險事件。

## Phase 2：每日工作流

4. 期限與風險
   整合待辦、行事曆、法扶長期未結與逾期風險，依嚴重度排序。

5. 對外資料
   依案件/案由產出可複製的當事人待補資料清單與說明文字。

6. 文件證據時間線
   從 `document_index` 建立案件文件時間線，供書狀與附件清單引用。

## Phase 3：諮詢與當事人互動

7. 諮詢／接案追蹤
   記錄潛在案件、諮詢摘要、聯絡方式與下一步，並同步跑利益衝突檢查。

8. 利益衝突檢查  
   查詢既有案件、當事人與對造資料；對造候選視為高風險，當事人候選需人工確認。

9. 當事人入口
   本期先提供可複製的安全入口內容；真正公開上傳入口保留設計但預設不開啟。

## Phase 4：管理視角

10. 業務統計
    不含計時，統計進行中案件、已結案、待辦、逾期待辦、文件索引、實務見解、法扶案件、學習與諮詢事件。

## 目前落地入口

- 前端：Paperclip `業務概覽` 內的 `管理工具`
- API：
  - `GET /api/osc/saas/overview`
  - `POST /api/osc/saas/conflict-check`
  - `POST /api/osc/saas/intake`
  - `POST /api/osc/saas/quality-check`
  - `POST /api/osc/saas/client-packet`
  - `GET /api/osc/saas/timeline`

## 與既有功能的整合原則

`管理工具`不是第二套案件系統，而是跨模組整理區：

- 期限與風險：讀取 `待辦事項`、`行事曆`、`法扶管理`。
- 利益衝突：讀取 `當事人`、`對造`、`案件列表`。
- 文件證據時間線：讀取 `書狀索引` 的 `document_index`。
- 修正學習中心：讀取 `AI 草擬` 的人工改正紀錄，跨案套用仍受案由限制。
- 對外資料：讀取案件資料與應備事項，輸出可複製文字；正式案件資料仍回 `案件列表` / `法扶管理` 維護。
- 高風險紀錄：讀取 `系統設定` 內的活動紀錄，只開刪除、歸檔、搬移、匯出、分享等高風險稽核。
- 業務統計：同時顯示 `案件總數`、`進行中案件`、`已結案`、`報結/送出中`，避免把總案件數和進行中案件混在一起。

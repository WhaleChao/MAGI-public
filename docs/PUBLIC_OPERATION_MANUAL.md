## 1. 版本定位

公開版 MAGI 是單一主機可自行安裝的法律事務輔助系統，重點是：

- 統一管理案件、待辦、行事曆、檔案與書狀草擬。
- 透過本機或自管模型進行摘要、翻譯、分類、OCR 輔助與檢索。
- 讓使用者保有資料主控權，敏感資料預設留在自己的主機與資料庫。
- 提供可檢查的健康狀態、安裝精靈、公開版隔離稽核與商用上線檢核。

公開版不是多租戶雲端服務。不同主機就是不同 MAGI，各主機需分別設定資料庫、模型、檔案空間、OAuth、通訊頻道與備份策略。

公開版預設不啟用下列項目：

- 原作者私有案件資料。
- 私人 NAS 路徑或私人雲端硬碟。
- 私有實務見解來源。
- 私人通訊頻道 webhook。
- 任何真實金鑰、密碼、cookie、OAuth token。

## 2. 系統需求

建議環境：

- macOS 或 Linux 主機。
- Python 3.12 以上。
- MariaDB 或相容資料庫。
- 可用磁碟空間至少 50 GB；若要大量 OCR、判決資料或檔案索引，建議 200 GB 以上。
- 可穩定連線的模型服務，例如 MLX sidecar、OpenAI 相容 API，或其他本機推論服務。
- 若使用 Google Calendar、Google Drive 或通訊軟體，需自行建立對應 OAuth 或 webhook。

安裝前請先確認：

- 你能在終端機執行 `python3 --version`。
- 你能建立本機 virtualenv。
- 你知道資料庫主機、帳號與資料庫名稱。
- 你已決定檔案存放位置，例如本機資料夾、NAS 掛載點或雲端同步資料夾。

## 3. 第一次安裝

公開版建議使用客戶安裝精靈。

```bash
git clone https://github.com/WhaleChao/MAGI-public.git
cd MAGI-public
python3 scripts/customer_install_wizard.py --public --yes
```

安裝精靈會做以下事情：

- 建立本機 `.env`。
- 產生本機用 secret。
- 檢查 Python、資料庫、檔案路徑、模型服務與必要套件。
- 安裝必要依賴。
- 建立本機 runtime 目錄。
- 寫出 `.runtime/customer_install_wizard_latest.json` 作為檢查紀錄。

安裝精靈不會把 token 或密碼列印到畫面，也不會要求你把 `.env` 提交到 git。

如果你只想先檢查環境，不要真的安裝：

```bash
python3 scripts/customer_install_wizard.py --public --no-live
python3 scripts/first_run_setup.py --public --json
python3 scripts/magi_doctor.py --json
```

## 4. 啟動、停止與健康檢查

啟動 MAGI：

```bash
./scripts/magi_cli.sh start
```

查看狀態：

```bash
./scripts/magi_cli.sh status
```

重啟：

```bash
./scripts/magi_cli.sh restart
```

停止：

```bash
./scripts/magi_cli.sh stop
```

瀏覽器開啟：

- 主頁：`http://127.0.0.1:5002/`
- 健康狀態：`http://127.0.0.1:5002/health`
- NERV 狀態頁：`http://127.0.0.1:5002/dashboard/nerv`

健康狀態應至少確認：

- 主狀態為正常。
- 資料庫正常。
- 推論服務正常。
- OCR 正常或已明確設定為停用。
- 向量資料庫正常。
- 檔案儲存位置可讀寫。
- 日常稽核最近有更新。

`/openclaw` 是已退役舊入口，公開版預期會回 404。請使用 MAGI 主頁與 `/magi-adjust`。

## 5. 首次設定

第一次登入後，請依序完成：

1. 設定管理員帳號與密碼。
2. 設定資料庫連線。
3. 設定案件檔案根目錄。
4. 設定模型服務。
5. 設定 OCR 模式。
6. 設定 Google Calendar 或其他行事曆來源。
7. 設定通訊頻道。
8. 執行健康檢查。
9. 執行公開版隔離稽核。
10. 執行商用上線檢核。

`.env` 只應留在本機，請勿提交。若要交付給客戶，請提供 `.env.example` 或安裝精靈，不要提供真實 `.env`。

## 6. MAGI 主頁與基本操作

MAGI 主頁提供主要功能入口：

- 案件管理：案件清單、狀態、資料夾、檔案與案件資訊。
- 業務概覽：待辦、行事曆事件、OSC 建立事件、近期案件提醒。
- 法扶管理：法扶案件、開辦、進度回報、結案與文件檢查。
- 書狀與範本：書狀索引、草擬、校對、範本資料夾。
- 檔案處理：預覽、下載、分享連結、OCR、命名。
- AI：問答、摘要、翻譯、法律資料查詢與工具調用。
- NERV：系統健康、排程、服務狀態與維運提示。

建議每日先看 `/health` 或 NERV 狀態頁，再進入案件作業。若健康狀態是需檢查，先處理警示，再進行批次下載、批次搬檔或正式送出。

## 7. AI 問答與工具調用

使用 AI 問答時，請用具體問題描述你要 MAGI 做的事：

- 「摘要這份 PDF 的主張與爭點。」
- 「把這份英文契約翻譯成繁體中文。」
- 「查詢這個案件的下一個行事曆事件。」
- 「找出這個當事人的待辦事項。」
- 「搜尋與消費者債務清理相關的內部資料。」

公開版的工具調用原則：

- 問行程時，應調用行事曆或待辦工具。
- 問檔案時，應調用案件檔案或索引工具。
- 問法律資料時，應調用知識庫或法律資料工具。
- 查不到時應明確回覆查不到，不應臆測。
- 高風險動作，例如正式送出、刪除、批次搬移，應保留確認步驟。

摘要、翻譯、逐字稿有交付前品質閘門：

- 摘要若偏題、過短、把文件誤判成案件建檔問答，會被阻斷或改用可追溯的抽取式摘要。
- 翻譯若只有開場語、相對原文過短、段落失敗或輸出工具推理，會被阻斷。
- 逐字稿若轉錄失敗、空白、過短或輸出工具推理，會被阻斷。
- 法律資料輸出會標示來源與覆核註記；未連結來源的模型知識不得直接引用。

相關設計見 [法律 AI 交付護欄](LEGAL_AGENT_GUARDRAILS.md)。

管理者可用測試命令確認工具調用：

```bash
python3 scripts/ops/commercial_readiness_live.py --strict-public
```

## 8. 案件管理

案件管理的核心資料包含：

- 案件編號。
- 當事人。
- 案由。
- 案件類型。
- 法院案號。
- 狀態。
- 資料夾位置。
- 待辦與行事曆。
- 檔案清單。

建議命名規則：

```text
YYYY-NNNN-當事人-程序或類型-案由
```

例如：

```text
2026-0001-王小明-一審-損害賠償
```

同名當事人可能有不同程序或不同案件，不能只用姓名判斷案件。MAGI 會優先使用案件編號、法院案號、法扶案號、案由與資料夾位置交叉確認。若仍無法判斷，應標示待確認，不應自動混併。

## 9. 檔案、PDF、OCR 與命名

檔案頁可以進行：

- 預覽。
- 下載。
- 開啟案件資料夾。
- 產生分享連結。
- OCR。
- PDF 命名。
- 歸檔。

PDF 命名流程會盡量避開掃描信封頁。法院通知、程序裁定、判決、對造書狀等檔案常有第一頁或第二頁為信封，MAGI 會先判斷是否為封面或信封，再抓取實質內容。

OCR 建議原則：

- 有文字層的 PDF 優先使用文字層。
- 掃描 PDF 再使用 OCR。
- OCR 結果需保留原始檔，不覆蓋原檔。
- 命名信心不足時，應進入人工確認或學習流程。

使用者修正命名後，MAGI 可記錄修正差異，後續同類型文件會優先套用相同規則。但不同案由、不同文件類型不可混用學習結果。

## 10. 書狀草擬與校對

書狀功能的建議流程：

1. 選擇案件。
2. 選擇案由與書狀類型。
3. 選擇範本。
4. 匯入案件事實、爭點與證據。
5. 產生草稿。
6. 由使用者校對。
7. 匯出 Word。
8. 視需要匯出 PDF。
9. 將完成版回存案件資料夾。

注意事項：

- MAGI 產生的是草稿，不是律師最終意見。
- 法律引用需由使用者確認。
- Word 排版應以範本樣式為準，不應破壞頁首、頁尾、字型、行距與編號。
- 使用者可把修改後版本回饋給 MAGI，作為同案由、同書狀類型的學習資料。

## 11. 行事曆與待辦

MAGI 支援兩類待辦：

- OSC 建立事件：由 MAGI 或 OSC 規則建立的案件待辦。
- 行事曆事件：由 Google Calendar 或其他行事曆來源匯入。

兩者應分開顯示，避免使用者把手動登錄的行程誤認為 OSC 自動待辦。

行事曆匯入建議：

- 每 6 小時更新一次。
- 只匯入可辨識案件編號或符合法扶計數規則的事件。
- 手動私人行程不應進入案件統計。
- 法扶需計數的會議、開庭、調解、聯繫、閱卷與律見可依規則匯入。

已完成事件應能標示完成。大量過期事件可用批次規則處理，但不可影響法扶報結與回報統計。

## 12. 法扶與外部機關流程

公開版可提供法扶流程管理，但每個使用者必須自行設定帳號、權限與資料來源。

法扶常用流程：

- 新派案建立。
- 開辦資料整理。
- 進度回報。
- 逾期提醒。
- 官網附件下載。
- 結案文件檢查。
- 結案資料夾搬移。

高風險原則：

- 正式送出前必須人工確認。
- 同名不同案不得只靠姓名自動結案。
- 已結案案件仍應可從 MAGI 開啟資料夾與檔案。
- 強制執行案件可能以執行命令作為結案依據，需依案件類型判斷。
- 若資料不完整，應標示待確認，不應硬送出。

## 13. 法律資料與知識庫

公開版支援法律資料索引與檢索。可使用：

- 本機文件。
- 公開資料集。
- 使用者自行匯入的判決或行政資料。
- 可公開使用的法律資料 MCP 或 API。

知識庫維護原則：

- 程序性文書、抓不到實質見解的資料、提示詞外洩內容，不應進入高信任知識庫。
- 重複向量應定期去重。
- degraded 或 warn 類資料應進入修復流程。
- 查不到資料時，AI 應明確說查不到。
- 法律資料需標示來源、日期與可信度。

## 14. 對外資料、預覽與分享

對外資料功能用於整理可提供給客戶、同事或外部對象的文字與檔案。

建議使用方式：

- 先在案件資料夾確認檔案正確。
- 用 MAGI 預覽檔案。
- 產生分享連結或複製對外文字。
- 用既有通訊管道傳送。
- 回到 MAGI 記錄已傳送或已回覆。

若檔案預覽或分享失敗，請先看：

- 檔案是否仍存在。
- 檔案是否正在被 Word、PDF 編輯器或同步程式鎖定。
- NAS 或雲端同步是否中斷。
- `/health` 的檔案儲存狀態是否正常。

## 15. 通知與通訊頻道

公開版支援通知分流。建議至少分成：

- 系統維運通知。
- 法扶一般通知。
- 法扶派案通知。
- 進度回報提醒。
- 檔案下載通知。
- 錯誤與需人工介入通知。

分流原則：

- 系統健康檢查不應送到業務頻道。
- 進度回報可送到需要處理的人。
- 新派案與一般巡檢不要混在一起。
- 高風險錯誤應有明確處理建議。
- 通知內容不要包含 token、cookie、密碼或完整私密資料。

## 16. 維運、備份與清理

建議每日：

- 查看 `/health`。
- 查看 NERV 狀態頁。
- 確認資料庫連線。
- 確認模型服務。
- 確認檔案儲存位置。
- 查看最新錯誤通知。

建議每週：

- 執行商用檢核。
- 檢查備份是否成功。
- 清理可刪快取。
- 檢查磁碟空間。
- 檢查重複向量與 degraded 知識項。

可清理項目通常包含：

- 暫存 OCR 檔。
- 已過期 runtime report。
- 可重建的模型快取。
- 可重建的下載快取。
- 測試輸出。

不可直接清理項目：

- `.env`。
- OAuth token。
- DB dump。
- 使用者案件檔案。
- 使用者自行建立的 JSON、SQLite、pickle 或單機資料檔。
- 尚未確認已備份的上傳、下載與歸檔資料。

## 17. 安全與隱私

公開版交付前請確認：

- `.env` 未被 git 追蹤。
- token、cookie、密碼、OAuth 憑證未被 git 追蹤。
- 私人路徑與私人信箱未被寫入公開文件。
- 私有資料來源未被公開版啟用。
- 使用者資料有備份與刪除策略。
- 對外分享連結有權限與期限控管。

公開版隔離檢查：

```bash
python3 scripts/public_release_audit.py --public-isolation --strict
```

若只是測試公開安裝性，且沒有私有 DB：

```bash
python3 scripts/public_release_audit.py --public-isolation --strict --skip-db
```

正式商用環境不得跳過 DB 與 live gate。

## 18. 商用上線檢核

正式交付前，請通過：

```bash
python3 scripts/public_release_audit.py --public-isolation --strict
./venv/bin/python scripts/ops/commercial_readiness_live.py --strict-public
```

檢核內容應包含：

- 安裝精靈可完成。
- MAGI daemon 可啟動。
- 主頁可開啟。
- `/health` 正常。
- 資料庫正常。
- 推論服務正常。
- OCR 正常或已明確停用。
- 檔案預覽、下載、分享正常。
- 行事曆匯入正常。
- 待辦分流正常。
- 通知分流正常。
- 高風險操作保留確認步驟。
- 公開版隔離檢查沒有 error 或 warning。

## 19. 常見問題

### 19.1 主頁打不開

請先確認：

```bash
./scripts/magi_cli.sh status
./scripts/magi_cli.sh restart
```

再開啟：

```text
http://127.0.0.1:5002/health
```

### 19.2 健康狀態顯示需檢查

請點入 NERV 狀態頁，看是哪一個服務異常。常見原因是模型服務未啟動、資料庫未連線、檔案儲存位置未掛載、排程太久未執行。

### 19.3 AI 回答與問題不符

請檢查工具調用是否正確。問行程應查行事曆，問檔案應查檔案索引，問法律資料應查知識庫。若沒有來源，應要求 MAGI 回覆查不到。

### 19.4 檔案預覽失敗

常見原因是檔案不存在、檔案被鎖定、同步尚未完成、NAS 中斷或權限不足。先確認原檔可讀，再重試預覽。

### 19.5 通知送錯頻道

請檢查通知分流設定。系統健康通知、法扶一般通知、法扶派案通知與進度回報通知應分開。

### 19.6 公開版稽核失敗

請先讀錯誤訊息，不要硬推。常見原因是 `.env` 被追蹤、文件出現私人路徑、測試資料含真實案件資訊、或私有資料來源未拔除。

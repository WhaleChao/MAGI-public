# MAGI 法律 AI 交付護欄

本文件記錄 MAGI 參考 Anthropic `claude-for-legal` 後落地的公開安全設計。參考來源：<https://github.com/anthropics/claude-for-legal>。

## 核心原則

1. 法律 AI 的輸出是草稿或輔助分析，不是律師審查的替代品。
2. 裁判、法規、實務見解、外部資料庫結果必須能回到來源。
3. 查不到就回覆查不到；不能用模型記憶補出不存在的判決、法條或事實。
4. 對外送出、引用或提交前，必須保留覆核註記。
5. 摘要、翻譯、逐字稿如果偏題、拒絕、過短或洩漏工具推理，必須被品質閘門阻斷。

## 來源標記

- `[本機資料庫]`：MAGI 已索引的裁判、文件或案件資料。
- `[使用者文件]`：本次上傳或指定的文件。
- `[法律 MCP]`：外部法律資料庫 MCP 回傳資料。
- `[公開網頁]`：公開網頁或官方網站。
- `[模型記憶，需核對]`：未連結來源的一般模型知識，不得直接引用。

## 覆核註記

法律工作流輸出應標示：

- 來源數量。
- 讀取範圍。
- 法律時效或版本是否已查核。
- 待核對事項。

## 三類文件產出品質閘門

摘要：

- 阻斷空白輸出、拒絕讀取、案件建檔問句、過短摘要、工具推理洩漏。
- 大型文件摘要失敗時，改用可追溯的抽取式備援摘要。

翻譯：

- 阻斷只有開場語、相對原文過短、偏題、工具推理洩漏。
- 段落失敗時不可假裝完整，必須標出重跑需求。

逐字稿：

- 阻斷轉錄失敗訊息、空白輸出、相對音訊過短、工具推理洩漏。
- 若產出檔案，檔案路徑與可下載 artifact 仍要通過同一回覆檢查。

## MAGI 對應程式

- `api/legal_workflow.py`：法律代理、來源標記、覆核註記、法律引用檢查。
- `api/handlers/output_quality_handler.py`：摘要、翻譯、逐字稿共用品質閘門。
- `api/handlers/summary_handler.py`：摘要模型輸出與抽取式備援。
- `api/handlers/translation_handler.py`：翻譯最終輸出品質阻斷。
- `api/domains/multimedia_flow.py`：音訊逐字稿、語音翻譯、語音摘要與上傳檔案流程。
- `api/blueprints/web_runtime.py`：網頁版檔案上傳直接任務路由。

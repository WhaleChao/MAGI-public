- `找關於詐欺的判決`

## 怎麼加入 MAGI（LINE / Telegram / Discord）

先講結論：目前這份程式碼裡，**沒有存放對外公開的 LINE 加好友連結、Telegram Bot username，或 Discord 伺服器邀請連結**。所以這裡先寫「正確加入方式」與「管理者要補什麼資訊」。

### LINE 加好友

一般使用者最常見的 3 種加入方式：

1. 掃管理者提供的 **LINE 官方帳號 QR code**
2. 點管理者提供的 **LINE 加好友連結**（通常會是 `line.me/...`）
3. 用管理者提供的 **LINE ID / 官方帳號名稱** 在 LINE 內搜尋

如果你是一般同事，最簡單做法就是直接跟管理者要：

- LINE 加好友連結
- LINE QR code

如果你是管理者，現行部署文件顯示 LINE 是主要對話入口之一；第一次管理員綁定時，還可能需要把系統產生的 `MAGI-XXXX` 綁定碼傳給 LINE Bot，讓 MAGI 知道你是管理者。

### Telegram 加好友

Telegram 通常不是掃 QR code，而是以下兩種方式：

1. 在 Telegram 搜尋 **MAGI 的 bot 使用者名稱**
2. 直接打開 **`https://t.me/<bot_username>`**

如果 MAGI 是要在群組裡使用，通常做法是：

1. 先把 MAGI Bot 加進群組
2. 再由群組管理員做通知綁定或 Topic 綁定

目前這份 repo 只有 Telegram Bot Token 類型的設定，**沒有公開寫出實際對外的 bot username**，所以正式使用時還是要由管理者補上對外帳號名稱。

### Discord 加入方式

Discord 跟 LINE / Telegram 不太一樣。大多數情況下，不是「加好友」，而是以下兩種其中之一：

1. **加入已經有 MAGI 的 Discord 伺服器**
2. **由管理者把 MAGI Bot 邀請到你們自己的伺服器**

如果你只是一般使用者：

- 直接向管理者索取 Discord 伺服器邀請連結
- 進入後到指定頻道和 MAGI 互動

如果你是管理者，要把 MAGI Bot 加進自己的伺服器，現行做法是：

1. 進 Discord Developer Portal
2. 到 `OAuth2` -> `URL Generator`
3. 勾選 `bot`
4. 勾選需要的權限
5. 複製產生的 invite URL
6. 用這個連結把 Bot 邀請進你的伺服器

如果你已經知道 Bot ID，也可以組成這種邀請格式：

- `https://discord.com/api/oauth2/authorize?client_id=<bot_id>&permissions=8&scope=bot`

目前 repo 內只有「怎麼建立 / 邀請 Discord Bot」的流程文件，**沒有放現成可對外公開的 Discord 伺服器邀請連結或固定 bot invite URL**。

## 3. 附件怎麼用

### 上傳檔案後最常用的 4 句

- `翻譯`
- `摘要`
- `不要摘要`
- `回到剛剛那份`

### 目前支援的檔案格式

- 翻譯：`PDF`、`EPUB`、`TXT`、`MD`、`LOG`、`CSV`、`JSON`、`DOCX`
- 摘要：`PDF`、`EPUB`、`TXT`、`MD`、`LOG`、`CSV`、`JSON`、`DOCX`
- 語音：`MP3`、`WAV`、`M4A`、`FLAC`、`OGG`、`WEBM`

### 直接指定本機檔案時的寫法

如果你不是上傳檔案，而是直接在訊息裡指定路徑，請用「絕對路徑」：

- `翻譯檔案 /Users/ai/Desktop/test.pdf`
- `掃描檔命名 /Users/ai/Desktop/scan.pdf`
- `逐字稿 /Users/ai/Desktop/audio.m4a`

| 你可以這樣說 | 會產生什麼 | 模式 |

## 5. 法扶（LAF）

> 安全預設：法扶流程一律先做草稿、預覽或填表，不直接送出。

### 最常用說法

| 你可以這樣說 | 作用 | 模式 |
|------|------|------|
| `法扶回報指令` | 顯示法扶回報說明與範例 | ⚡ |
| `幫我做蕭仁俊開辦回報` | 開辦填表 | 🔄 🔒 |
| `幫[當事人F]做疑義回報 原因 資力不合標準` | 疑義回報 | 🔄 🔒 |
| `[當事人E]訴訟中費用支付回報 說明 裁判費2000` | 訴訟中費用支付 | 🔄 🔒 |
| `[當事人L]二階段回報` | 二階段 / 調解不成立回報 | 🔄 🔒 |
| `蔡旭欽結案回報 原因 本案無閱卷必要` | 結案回報 | 🔄 🔒 |
| `[當事人F]受扶助人撤回回報 原因 申請人撤回` | 撤回回報 | 🔄 🔒 |
| `法扶回報 疑義 1140728-K-002 原因 資力不合標準` | 結構化寫法 | 🔄 🔒 |
| `法扶監控` / `派案監控` | 監控法扶信件 / 派案 | 🔄 |
| `自動報結掃描` | 批次掃描待報結案件 | 🔄 |
| `二階段草稿批次` | 批次產生二階段草稿 | 🔄 |

### 重要規則

- 開辦不會直接送出。
- MAGI 會先填好、截圖給你看。
- 若要真的送出，請回：`正確送出 <確認碼>`
- 若要取消，請回：`取消送出 <確認碼>`

### 可以用什麼當目標

- 當事人姓名，例如 `蕭仁俊`
- 法扶案號，例如 `1140728-K-002`
- 案件系統編號，例如 `2026-0013`

## 18. 注意事項

- 法扶送出類流程預設不直接送出。
- 閱卷、筆錄、法扶、資料庫、夜間任務這類流程通常是背景執行，會稍後主動回報。
- 若你是用「指定路徑」的方式叫 MAGI 讀檔，路徑請盡量用絕對路徑。
- 若你不確定 MAGI 會把一句話當成什麼功能，先用：`查詢路由 [你的句子]`
- 若你不是管理員，`🔒` 類功能可能會被拒絕，或改成通知管理員審批。

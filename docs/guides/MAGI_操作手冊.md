# MAGI 操作手冊

> 版本：2026-04-25 重寫版
> 系統現況：72 個技能模組、53 條定時任務、三入口（LINE / Telegram / Discord）全綠
> 適用對象：律師本人 + 受邀的事務所同仁
> 全文採台灣用語

## 0. 三十秒上手

MAGI 不是 CLI 工具，**請用白話中文說**：

| 你想做的事 | 白話說法 |
|---|---|
| 翻譯一份英文判決 | 上傳 PDF，回「翻譯」 |
| 幫劉先生打報價單 | `開報價單 劉信義 答辯狀代擬 12000` |
| 幫蘇先生申請閱卷 | `閱卷聲請 高院 114重上更二95 蘇先生` |
| 看下週要開幾場庭 | `下週行程` |
| 民法第 184 條三要件 | 直接打字問即可 |
| 想用更強模型 | `@heavy 問題內容` |

LINE / Telegram / Discord 三邊任一邊都通，內容會同步進系統記憶。

## 1. 怎麼把 MAGI 加進來

### 1.1 LINE
- 在 LINE 上加好友：搜尋官方 ID `@magi-law`（管理員提供）
- 直接傳訊息即可，不需 `/` 前綴
- 圖片、PDF、語音檔可直接傳

### 1.2 Telegram
- 找機器人：搜尋 `@MAGI_CASPER_01_666_BOT` 或管理員給的連結
- 第一次傳訊息要先 `/start`
- Telegram 群組可建立主題（Topic），用 `綁定主題 <名稱>` 把對應通知釘在該主題

### 1.3 Discord
- 邀請連結請向管理員索取
- Bot 名稱：`MAGI`
- Discord 預先設好分類頻道（法扶、閱卷、市場簡報、法院通知等），訊息會自動分流到對應頻道
- 在法扶子頻道（如 `laf-progress`）只輸入人名或案號，會自動補成對應指令

### 1.4 三個都加好友的好處
- 寫文書、查判決：在電腦前用 Discord
- 出庭時用 LINE 快速回報
- 看晨間摘要：Telegram 訊息排版最乾淨

## 4. 附件（PDF / 檔案）

### 4.1 上傳後最常用的 4 句
1. `翻譯` — 整份翻
2. `摘要` — 重點整理
3. `辨識` — OCR
4. `歸檔` — 自動命名 + 放進 NAS 對應案件資料夾

### 4.2 支援格式
| 類型 | 副檔名 |
|---|---|
| 文件 | `.pdf .docx .doc .pptx .xlsx .txt .md .html .epub` |
| 圖片 | `.png .jpg .jpeg .heic .webp` |
| 語音 | `.mp3 .m4a .wav .aiff .ogg .flac` |

### 4.3 自動歸檔規則（PDF Namer）
傳一份判決書 → MAGI 會：
1. Apple Vision OCR + Tesseract 共識，抓出 法院 / 案號 / 當事人 / 文件類型 / 期限
2. 命名：`YYYYMMDD 法院全名+案號+文件類型（當事人；N日內XX）.pdf`
3. 偵測到「N 日內 XX」會自動建立 OSC 待辦事項並推 Discord 通知

## 5. 法扶（LAF）

### 5.1 自動處理（不用做事）
- 法扶基金會的派案信進信箱 → MAGI 自動建立案件、編號、新增當事人、建立 NAS 案件資料夾
- 進度回報期限到期 → 自動產生草稿 + 確認碼，律師回確認碼即送出

### 5.2 手動標記
| 指令 | 用途 |
|---|---|
| `[姓名] 已開辦` / `[案號] 已開辦` | 標記開辦完成 |
| `[姓名] 已報結` / `[案號] 已報結` | 標記結案 |
| `法扶監控` | 看所有案件當前狀態 |
| `法扶回報指令` | 顯示完整回報格式 |

> Discord 法扶子頻道支援「只輸入人名或案號」，會自動補對應回報類型。

### 5.3 OSC 法扶工作台
- 位置：開啟 OSC → `法扶` 頁籤 → 左側選案件 → 右側案件詳情。
- 消債案件：在右側工具列按 `開啟/編輯應備事項表`，會出現 OSC 原本的消債應備文件確認表。
- 勾選當事人狀況後，系統會自動展開對應文件，例如扶養父母、扶養子女、租屋、居住親屬房產、訴訟/強執、協商/調解、五年內營業、存摺無法補登、無薪資單、自訂項目。
- 狀態使用 `待補`、`已繳`、`免附`；`已繳` 與 `免附` 不會再被列為待補件。
- 按 `產生可複製文字` 可輸出要傳給當事人的待補文件文字。
- 按 `自動帶入/生成字號` 會從案件資料夾中的開辦通知書、接案通知書、准予扶助證明書、委任狀等檔名偵測法扶字號；若有多個候選字號，請先手動填入正確字號後再按一次。

### 5.4 二階段確認碼
所有法扶 portal 送出（開辦、進度、結案）都採兩階段：
1. MAGI 在 portal 把表單填好 → 暫存（不送出）
2. 截圖傳到 Discord，附 6 位確認碼（30 分鐘內有效）
3. 律師檢查無誤後回 `正確送出 ABCDEF`，MAGI 才真正送出
4. 不要送回 `取消 ABCDEF`

### 5.5 已遞委任 / 法扶模式（閱卷時）
| 句尾加詞 | 行為 |
|---|---|
| `已遞委任` | 略過委任狀附件，只填表 |
| `法扶` | 只附法扶通知書，不附委任狀 |

## 6. 閱卷

### 6.1 三大常用句
```
閱卷查核 <法院簡稱> <年度+案號>
閱卷聲請 <法院簡稱> <年度+案號> <當事人>
下載閱卷 <法院簡稱> <年度+案號>
```
範例：
```
閱卷查核 高院 114重上更二95
閱卷聲請 花蓮 115原訴36 陳文明
閱卷聲請 花蓮 115原訴36 陳文明 已遞委任
閱卷聲請 宜蘭 115原訴36 陳文明 法扶
```

### 6.2 繳費憑證最穩做法
1. 律師收到繳費通知（Discord `filereview-payment` 頻道）
2. 在臨櫃或網銀繳費後，**直接把繳費收據圖傳到 Discord**
3. MAGI 自動辨識金額、案號、繳費時間，回上傳 portal
4. portal 回應後寄送收據到對應頻道

### 6.3 兩階段確認碼（與法扶相同）
閱卷聲請、繳費憑證上傳都走兩階段流程，不主動送出。

### 13.1 自動運維

- **每週日 04:00** 清理 Vision / HuggingFace cache（atime > 14 天）
- **每 30 秒** memory_watchdog 監控（swap > 8 GB 或可用記憶體 < 2 GB 觸發）
- **每分鐘** Cron 排程檢查（53 條任務）

### 13.2 自我修復
失敗的 cron 會自動寫進 `.runtime/issue_agenda.jsonl`：
- PII 自動遮蔽（身分證、手機、API key）
- 5 分鐘去重，5000 筆自動輪替
- Telegram 自癒頻道每日推送摘要（綁定主題：`綁定主題 自癒頻道`）

## 15. 三模型架構（背景知識）

MAGI 用三個本機模型分工，自動依時段切換：

| 時段 | 主推理 | 邏輯審查 | 格式稽核 |
|---|---|---|---|

| **夜間** 21:50–07:00 | Gemma-4 26B | — | — |

## 16. 隱私與資料安全

### 16.1 哪些資料留在本機
- 所有當事人姓名、案號、法扶 ID、文件
- NAS 上的卷證、書狀、判決
- MAGI 對話歷史 SQLite
- 系統失敗紀錄、debug 截圖

### 16.2 哪些資料會送到外網
- `@heavy` 指令會送 NVIDIA NIM；姓名等個資先遮蔽
- 司法院判決搜尋走司法院公開站
- 判決爬蟲走司法院 OpenData
- DuckDuckGo / Google 翻譯（短文）

### 16.3 不會送出網的
- 法扶 portal 帳密（只在本機 Playwright session 用）
- 閱卷 / 筆錄 portal 帳密（同上）
- LINE / Telegram / Discord token（只本機呼叫官方 API）
- Google Calendar OAuth token

### 16.4 GitHub repo 隱私狀態
- 倉庫設為 Private
- 歷史已清掉所有當事人姓名 + 業務記憶
- `.runtime/`、`CLAUDE.md`、`cron_jobs.json` 在 `.gitignore`，不會推送

## 17. 故障時的處置

### 17.1 MAGI 沒回應
1. LINE 沒回 → 等 5 秒（cold start）
2. 再沒回 → `MAGI 系統狀態`
3. 還是不行 → 看本機 menubar 圖示是否亮綠
4. 都不行 → `magi restart`（終端機）

### 17.2 法扶 portal 失敗
- 一般是密碼到期警告（已自動處理）
- 連 5 次失敗會自動通知管理員
- 通常 30 分鐘內法扶 portal 自己會恢復

### 17.3 閱卷網站登入失敗
- 多半是驗證碼 OCR 失敗
- 系統會自動重試 8 次
- 連續 3 次失敗會通知律師

### 17.4 oMLX 沒回應
- 凌晨自動切夜間模式（21:50）會短暫不可用
- 早上 07:00 切日間模式同樣短暫不可用
- `cool start` 後第一次推理會比較慢（10–20 秒）

## 18. 使用習慣建議

1. **不需要記指令格式** — 大多數指令都有自然語言別名
2. **附件先傳，再說要做什麼** — 比較順
3. **重要決定請律師親自確認** — 兩階段確認碼是給律師最後把關
4. **想要更精準** — 加 `@heavy`，但會比較慢（10–60 秒）
5. **想要紀錄想法** — 直接說 `記住 ...`，下次回想用

## 19. 一句話版

> **MAGI 像個會處理法務瑣事的事務所助理**：你用嘴巴交代任務，它記住、照辦、提醒你確認；不會的就說不會、不確定的就附原始連結讓你查；個資不離開本機，需要送雲端的會先遮蔽。

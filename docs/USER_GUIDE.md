# MAGI 一般使用者手冊

版本：2026-05-17
適用對象：一般使用者、事務協作者、導入顧問
適用版本：MAGI 公開版與私有版

本手冊以「每天實際怎麼用」為主，不要求使用者懂程式。公開版與私有版的基本操作相同；私有版可能額外啟用法扶入口、閱卷、筆錄、NAS、內部通訊頻道與私有法律資料來源。若你的畫面沒有某個功能，通常代表該功能尚未設定、權限不足，或該版本未開啟。

MAGI 是法律與事務作業的 AI 輔助系統，不會取代律師或專業人員的最終判斷。所有正式送出、刪除、批次搬移、公開分享、法律引用與金額計算，都應在送出前由使用者確認。

## 1. MAGI 能幫你做什麼

MAGI 會把案件、檔案、行事曆、待辦、法扶、書狀、摘要、翻譯與法律資料查詢集中在同一個系統中。你可以透過瀏覽器、LINE、Discord、Telegram 或其他已設定的通訊管道使用。

常見用途：

- 查案件資料、開啟案件資料夾、預覽或下載檔案。
- 讀取 PDF、OCR 掃描文件、建議檔名、建立待辦。
- 摘要判決、書狀、合約、研究文獻或會議資料。
- 翻譯文件，並保留法律用語、專有名詞與原文對照。
- 將錄音或影片轉成逐字稿，整理重點與待辦。
- 建立或查詢行事曆事件、OSC 建立待辦與法扶活動統計。
- 管理法扶案件的開辦、進度回報、文件檢查與結案流程。
- 搜尋判決、法條、知識庫、實務見解與指定關鍵字裁判。
- 產生書狀草稿、套用範本，並記錄使用者修正供同案由後續學習。
- 檢查系統健康、磁碟容量、模型狀態、OCR 與資料庫狀態。

MAGI 的回答如果沒有資料來源或工具回傳，就應該說明「查不到」或「資料不足」，不應直接猜答案。

## 2. 開始使用

### 2.1 瀏覽器入口

常見入口如下，實際網址請以你的安裝者提供的設定為準：

```text
http://127.0.0.1:5002/
http://127.0.0.1:5002/health
```

若系統有對外連線設定，可能會提供 Tailscale、Cloudflare 或其他遠端網址。外網打不開時，先確認本機 `/health` 是否正常，再檢查遠端通道。

### 2.2 登入

第一次使用時，管理者會提供登入方式。若系統要求設定密碼，請改成你自己的密碼，不要沿用預設密碼。

若忘記密碼，請找管理者重設。公開版不應內建任何真實帳號或金鑰。

### 2.3 第一次檢查

登入後先看健康狀態：

- 主狀態是否正常。
- 資料庫是否正常。
- 推論服務是否正常。
- OCR 是否正常。
- 檔案根目錄是否可讀寫。
- 行事曆、通知、NAS 或雲端硬碟是否已連上。

若狀態顯示「需檢查」，先處理警示，再做正式作業。

## 3. 主畫面與頁籤

不同版本的名稱可能略有差異，但主要功能通常包含：

- **案件列表**：查看案件卡片、案件狀態、案由、法院案號與資料夾。
- **業務概覽**：查看近期待辦、行事曆事件、OSC 建立待辦與重要提醒。
- **行事曆**：查看庭期、會議、調解、電話、補正期限與自動建立事件。
- **待辦事項**：查看要處理的期限、文件與工作。
- **法扶管理**：查看法扶案件、開辦、進度回報、應備文件與結案。
- **書狀**：查看書狀索引、草擬書狀、範本資料夾與修改紀錄。
- **檔案**：預覽、下載、分享檔案，或查看案件資料夾內容。
- **法律資料**：搜尋判決、法條、知識庫與實務見解。
- **MAGI 狀態**：查看模型、OCR、資料庫、排程與磁碟狀態。

畫面文字應以臺灣慣用語呈現，例如「業務概覽」、「案件列表」、「資料夾」、「對外資料」。若看到不自然或不清楚的用語，請回報給管理者。

## 4. 與 MAGI 對話

### 4.1 自然語言即可

你可以直接說：

```text
請幫我摘要這份 PDF
請把這份英文文件翻成繁體中文
查一下王小明案件最近有什麼待辦
搜尋最高法院關於通譯的判決並分類
幫我產生這件的開庭準備清單
```

MAGI 會判斷應該使用哪個工具。正確行為範例：

- 問行程，要查行事曆或待辦，不應回答天氣。
- 問天氣，才使用網路搜尋或天氣來源。
- 問案件，要查案件資料庫，不應用一般網路搜尋亂找。
- 問判決或法條，要查法律資料工具。
- 查不到時，要說查不到，不應編造。

### 4.2 `@heavy` 或高品質模式

較長、較重要或要交付給外部的人，可以要求高品質處理：

```text
@heavy 請翻譯這份 PDF，保留專有名詞原文，並產生中英對照表
@heavy 請詳細摘要這份監察院報告，列出事實、爭點、結論與可引用段落
```

高品質模式會比較慢，但會做更嚴格的品質檢查。若品質閘門不通過，MAGI 應重新處理或說明失敗原因，不應交付明顯不完整的成品。

### 4.3 需要你補資料時

如果資訊不足，MAGI 可能會問：

```text
請提供案件編號或當事人姓名。
請提供要翻譯的檔案。
同名案件不只一件，請指定法院案號或法扶案號。
```

這不是故障，而是避免把不同案件混在一起。

## 5. 案件管理

### 5.1 查看案件

進入「案件列表」後，可以用以下方式找案件：

- 案件編號。
- 當事人姓名。
- 法院案號。
- 法扶案號。
- 案由或關鍵字。
- 案件狀態。

案件卡片通常會顯示：

- 當事人。
- 案件編號。
- 案由。
- 案件分類。
- 程序或案件種類。
- 法院與法院案號。
- 狀態。
- 法扶標籤。

法扶案件的標籤應清楚顯示，例如「法扶／進行中」、「法扶／已結案」。只用顏色但沒有文字，容易誤解。

### 5.2 進行中、結案中與已結案

建議理解如下：

- **進行中**：案件仍在處理，資料夾通常留在進行中區。
- **結案中**：已進入結案流程，但還有文件、送出、費用或資料夾搬移要確認。
- **已結案**：案件完成結案，資料夾應移到已結案區，網頁仍能開啟與預覽。

若畫面出現「進行中/結案中」這種合併標示，代表分類不夠清楚，應請管理者修正。

### 5.3 同名不同案

同一當事人可能有不同程序或不同案件。MAGI 應以案件編號、法扶案號、法院案號、案由與資料夾位置交叉判斷，不應只靠姓名搬移、結案或統計。

如果 MAGI 顯示「找到多個候選」，請補充更精確資訊。

## 6. 檔案預覽、下載與分享

案件檔案頁應支援：

- 預覽 PDF、圖片、文字檔與常見文件。
- 下載檔案。
- 建立分享連結。
- 複製連結給同事或客戶。
- 開啟檔案所在資料夾。

如果看到：

```text
檔案操作沒有完成
檔案讀取失敗
找不到檔案
```

請先檢查：

1. NAS 或雲端硬碟是否已掛載。
2. 檔案是否剛搬移或同步中。
3. 檔案是否被 Word、PDF 編輯器或其他程式占用。
4. 外網通道是否正常。
5. 健康狀態是否有檔案服務警示。

MAGI 應盡量用穩定的檔案讀取方式處理網路硬碟，避免因檔案鎖定造成預覽、下載或分享失敗。

## 7. PDF、OCR 與檔名

### 7.1 PDF 命名

你可以說：

```text
請幫我命名這份 PDF
請批次整理這個資料夾的法院文件
```

MAGI 會嘗試辨識：

- 文件日期。
- 法院或機關。
- 案號。
- 當事人。
- 文件類型。
- 期限或待辦。
- 是否第一頁是信封或封面。

法院通知、程序裁定、判決、對方書狀常有信封或封面，MAGI 應排除干擾頁，再判斷真正文件內容。

### 7.2 OCR

掃描檔或影像 PDF 需要 OCR。公開版通常使用本機 OCR 或可設定的 OCR 引擎；私有版可能接更完整的 OCR 管線。

OCR 品質不足時，MAGI 應提示需人工複核，不應把亂碼當成正確內容建立待辦或命名。

### 7.3 從 PDF 建立待辦

如果 PDF 檔名或內容包含期限，例如「10 日內補正」、「訂 6 月 1 日下午 4 時調解」，MAGI 可以建立 OSC 待辦或行事曆事件。

若你沒看到新待辦，請確認：

- PDF 是否已進入案件資料夾。
- 自動掃描排程是否有執行。
- PDF 是否命名成功或 OCR 成功。
- 案件是否能被唯一識別。
- Google Calendar 或行事曆整合是否正常。

## 8. 摘要、翻譯與逐字稿

### 8.1 摘要

你可以要求：

```text
摘要這份 PDF
整理成三點
請列出事實、爭點、法院理由與結論
請做可以貼給客戶看的白話摘要
```

好的摘要應包含：

- 文件類型。
- 主要事實。
- 爭點。
- 理由或依據。
- 結論。
- 期限或下一步。
- 可引用段落或頁碼。

若摘要只有空泛開場、語言錯誤、把文件內容講錯，應重跑或回報。

### 8.2 翻譯

你可以要求：

```text
請翻譯成繁體中文
請產出中英對照表
請保留每個專有名詞的原文
請用法律文件語氣翻譯
```

MAGI 的高品質翻譯應做到：

- 段落順序不亂。
- 不漏段。
- 專有名詞後保留原文，例如「成癮者（addicts）」。
- 法律用語符合臺灣慣用語。
- 不把普通語句誤譯成奇怪語意。
- 產出檔案內保留術語表或專有名詞表。

如果你看到錯譯，例如語意明顯不通，請直接指出「這句應改成……」。MAGI 應記錄修正，供同類型翻譯品質改善。

### 8.3 逐字稿

你可以上傳音訊或影片，要求：

```text
請轉逐字稿
請整理會議紀錄和待辦
請把逐字稿整理成摘要
```

好的逐字稿應包含：

- 可讀的段落。
- 說話者或時間資訊，若來源可辨識。
- 不確定詞標示。
- 待辦與決議整理。

音訊品質太差時，MAGI 應提示需人工複核。

## 9. 書狀、範本與修正學習

### 9.1 書狀草擬

你可以說：

```text
幫我草擬起訴狀
請依這件案件資料草擬陳報狀
請用範本產生聲請狀
```

MAGI 應先讀取案件資料、已歸檔文件與書狀範本，再產生草稿。草稿應保留可修改空間，不應直接當成完稿送出。

### 9.2 範本資料夾

範本是特殊資料，不是一般案件。範本頁籤應能：

- 查看範本清單。
- 預覽範本。
- 下載範本。
- 分享範本連結。
- 複製給他人。

範本不應顯示一般案件狀態，也不應把測試用分類當成真實案由。

### 9.3 修正學習

如果你修改了 MAGI 產出的書狀，可將修改前後差異回饋給 MAGI。學習規則：

- 只套用在同案由、同文件類型。
- 不同案由不可混用。
- 不應覆蓋使用者明確指定的內容。
- 學習紀錄應可追溯、可清除。

## 10. 行事曆與待辦

業務概覽應分開顯示：

- **OSC 建立待辦**：由案件文件、期限、PDF 命名或系統規則建立。
- **行事曆事件**：由 Google Calendar、Apple Calendar 或外部行事曆匯入。

兩者不能混在同一個框裡，否則使用者會分不清是文件期限還是行程。

常見操作：

```text
今天有什麼事
下週有哪些庭
查某案件的待辦
把這件標示已完成
```

行事曆同步應避免把同事手動登錄的私人行程全部混入案件待辦。一般案件事件應有案件編號；法扶活動計數事件則需由 DB 確認是法扶案件。

## 11. 法扶作業

公開版可提供法扶流程框架；私有版若已設定帳號與權限，可能可直接處理實際案件。

常見功能：

- 新法扶派案建立。
- 官網附件下載。
- 開辦。
- 應備事項表。
- 消債文件確認。
- 進度回報。
- 結案。
- 已結案資料夾搬移。

重要原則：

- 開辦是正式流程，不應出現「只暫存」的舊流程混淆。
- 已結案案件搬移後，網頁仍應能開啟資料夾與檔案。
- 同名不同案件不可誤搬。
- 消債所得清單應依年度自動更新，例如民國 115 年 5 月後可列 113、114 年；下一年類推。
- 進度回報提醒若已回報，應可用「某案已回報」冷卻一段期間後再提醒。

## 12. 閱卷與筆錄

私有版若已設定司法院登入與下載流程，可以處理：

- 閱卷申請。
- 閱卷通知檢查。
- 可下載檔案下載。
- 到院閱卷通知整理。
- 筆錄下載與更名。
- 歸檔到案件資料夾。

閱卷去重原則：

- 已下載且已歸檔的案件，不應一直重複通知。
- 只有繳費單的日期資料夾，不應算成實際閱卷次數。
- 同案多列通知應合併呈現。

筆錄通知原則：

- 只列出有下載到檔案的案件。
- 不要列一長串 0 份案件造成閱讀負擔。

## 13. 法律資料、判決搜尋與通譯實證研究

### 13.1 判決與法條

你可以說：

```text
查民法第184條
搜尋最高法院關於背信罪的判決
找消費者債務清理的實務見解
```

MAGI 應使用法律資料工具或已設定的法律資料來源。查不到時應說查不到，不應用模型記憶補編。

### 13.2 通譯實證研究

你可以說：

```text
請用關鍵字「最高法院 通譯」上網抓取裁判並分類
請把最高法院提到通譯的判決整理成表格
請區分只是條文帶過、通譯品質爭議、未用通譯、外語證據翻譯與法院結果
```

MAGI 會：

1. 到司法院裁判系統搜尋指定關鍵字。
2. 抓取裁判全文。
3. 整理成 TXT。
4. 依通譯相關原文分類。
5. 輸出 CSV、XLSX、Markdown。
6. 保留來源 URL、裁判字號、法院結果與原文摘錄。

若你指定不同關鍵字，MAGI 應依你的命令抓取與分類，不應只讀本機既有資料夾。

## 14. 帳務與支出

若已啟用帳務功能，MAGI 可協助：

- 匯入帳務表。
- 按月整理收入與支出。
- 標記固定支出。
- 避免重複計算薪資、勞健保、退休金或其他固定支出。
- 產出現金流與成本摘要。

使用者仍需確認：

- 匯入來源是否正確。
- 哪些項目不屬於本人或本事務所。
- 固定支出金額是否已更新。
- 是否有重複列帳。

## 15. 通知與通訊軟體

MAGI 可以透過 LINE、Discord、Telegram 或其他管道通知。通知應分流：

- 系統健康與巡檢：系統通知頻道。
- 法扶派案：法扶派案頻道。
- 法扶進度回報：進度回報或指定頻道。
- 閱卷與筆錄：業務通知頻道。
- 股市或情報：對應的資訊頻道。

不應把系統巡檢、健康檢查或測試訊息送到業務頻道，以免干擾工作。

## 16. 健康狀態與日常維護

進入 `/health` 或 MAGI 狀態頁，至少確認：

- 主狀態。
- 資料庫。
- 推論服務。

## 17. 資料安全與隱私

使用 MAGI 時請遵守：

- 不把真實金鑰、密碼、cookie、token 貼到公開頻道。
- 不把私有案件資料推到公開版。
- 分享連結前確認對象與檔案內容。
- 正式送出前檢查附件是否正確。
- 批次刪除、批次搬移與 DB 還原前要備份。
- 客戶文件、案件資料與個資應依所在地法規與事務所規範處理。

公開版應只包含程式、範例設定與脫敏文件，不應包含私有資料。

## 18. 常見問題

### Q1：MAGI 查不到資料，可以叫它猜嗎？

不建議。法律與案件資料應以可驗證來源為準。MAGI 查不到時應回覆查不到，或請你補資料。

### Q2：檔案預覽或下載失敗怎麼辦？

先確認檔案是否存在、NAS 是否掛載、外網是否正常、檔案是否被其他程式占用。若 `/health` 顯示檔案服務異常，請先處理健康狀態。

### Q3：翻譯品質不好怎麼辦？

使用 `@heavy` 或明確要求「保留專有名詞原文」、「中英對照表」、「法律文件語氣」。若仍錯誤，指出錯句與正確譯法，讓 MAGI 記錄修正。

### Q4：行事曆事件和 OSC 待辦有什麼差別？

行事曆事件是時間行程，例如開庭、會議、電話。OSC 待辦是系統依案件或文件建立的工作，例如補正、上訴期限、資料待補。兩者應分框顯示。

### Q5：為什麼同名案件要我再確認？

因為同一人可能有多個程序或多個案件。MAGI 要避免把 A 案的結案、待辦或法扶統計誤套到 B 案。

### Q6：可以把 MAGI 對外給客戶用嗎？

可以，但應先完成公開版隔離檢查、安裝精靈、健康檢查、煙霧測試、資料脫敏、金鑰清查與操作手冊確認。不要把私有資料夾或 `.env` 交給客戶。

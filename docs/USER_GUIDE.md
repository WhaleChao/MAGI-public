## MAGI 是什麼

MAGI 是事務所內部的 AI 作業平台。你可以透過 **LINE**、**Discord**、**Telegram** 或 **網頁儀表板** 與 MAGI 溝通，它會自動理解你的需求並執行對應功能。

MAGI 能做的事包括：
- 自動閱卷、下載筆錄、歸檔案件資料
- 查判決、搜法條、計算加班費/資遣費
- 產生書狀、委任狀、存證信函、收據
- PDF 處理（合併、分割、OCR、重命名）
- 股市追蹤與每日晨報
- 網路搜尋與深度研究
- 翻譯、摘要、記憶筆記
- 系統健康監控與自動巡檢

你不需要記住所有指令——**用自然語言說就好**，MAGI 會自動理解。

## 如何與 MAGI 對話

### 最簡單的方式：直接說

MAGI 支援自然語言，你可以像跟人說話一樣跟它對話：

```
你：查一下詐欺的最高法院判決
MAGI：[自動搜尋並回傳判決清單]

你：幫我把這個 PDF 重新命名
MAGI：[自動 OCR 辨識內容，依日期/文件類型重命名]

你：月薪 35000 加班 3 小時多少錢
MAGI：[計算加班費並回覆結果]
```

### 用前綴指令（更明確）

如果你想更精確地觸發功能，可以加上前綴：

```
@MAGI 系統狀態
/搜尋 不當得利
!翻譯 This is a test
```

支援的前綴：`@MAGI`、`/`、`!`

### 輸入 `help` 或 `幫助` 查看功能選單

```
你：幫助
MAGI：[回覆完整功能選單，依角色顯示可用功能]
```

## 頻道總覽

| 頻道 | 怎麼用 | 特點 |
|------|--------|------|
| **LINE** | 直接傳訊給 MAGI Bot | 可傳 PDF/圖片附件、最常用的管道 |
| **Discord** | 在指定頻道輸入訊息或 @MAGI | 適合長文回覆、多人協作、查看排程通知 |
| **Telegram** | 傳訊給 MAGI Telegram Bot | 股市晨報自動推送、全球情報推送 |
| **Web 儀表板** | 瀏覽器開啟 `http://localhost:5002` | 案件管理系統（OSC）、系統監控 |

> **提示**：LINE 有 3 秒回應限制，較耗時的任務會先回覆「處理中」再在背景執行。

## 指令格式

MAGI 支援三種輸入方式，從最簡單到最精確：

### 方式一：自然語言（推薦）

直接用中文描述你的需求，MAGI 會自動辨識意圖：

```
查關於勞資爭議的判決
幫我翻譯這段英文
下週有什麼庭要開
```

### 方式二：關鍵字觸發

使用特定關鍵字直接觸發功能：

```
股市晨報
開庭準備
法扶報結
系統狀態
```

### 方式三：結構化指令

更精確的參數化指令：

```
/搜尋 關鍵字
/翻譯 要翻譯的文字
/摘要 要摘要的內容
```

## 完整功能一覽

### 1. 法務自動化

#### 閱卷（file-review-orchestrator）

| 你可以說… | MAGI 會做… |
|-----------|-----------|
| `閱卷` / `申請閱卷` | 啟動端到端閱卷流程 |
| `下載閱卷文件` | 從司法院下載已核准的閱卷資料 |
| `閱卷繳費` | 查詢並追蹤閱卷繳費狀態 |
| `歸檔閱卷` | 將下載的文件自動歸檔到 NAS 案件資料夾 |

> 閱卷流程包含：SSO 登入 → 驗證碼自動破解 → 申請 → 文件下載 → 繳費追蹤 → 歸檔。

#### 法扶作業（laf-orchestrator / laf-portal-automation）

| 你可以說… | MAGI 會做… |
|-----------|-----------|
| `法扶報結` | 啟動法扶結案流程 |
| `法扶結案` | 處理法扶案件結案（含活動次數計算、費用申請） |
| `法扶未開辦掃描` | 掃描待開辦的法扶案件 |

支援 6 種工作流程：結案、開辦、狀態報告、撤回、延長、費用請領。

#### 筆錄處理

| 你可以說… | MAGI 會做… |
|-----------|-----------|
| `下載筆錄` / `同步筆錄` | 自動從司法院下載電子筆錄 |
| `搜尋筆錄 [關鍵字]` | 用語意搜尋已索引的筆錄內容 |
| `索引筆錄` | 將筆錄向量化建索引（支援依發言人/日期/內容搜尋） |

## 常用操作範例

### 日常法務作業

```
你：查一下不當得利的最高法院判決
MAGI：找到 5 筆相關判決：
      1. 最高法院112年度台上字第1234號…
      2. …

你：月薪 42000 加班 2.5 小時多少錢
MAGI：加班費計算結果：
      前 2 小時：42000/30/8 × 1.34 × 2 = $469
      第 3 小時（0.5h）：42000/30/8 × 1.67 × 0.5 = $146
      合計加班費：$615

你：做委任狀
MAGI：好的，請告訴我以下資訊：
      1. 委任人姓名？
你：王大明
MAGI：2. 案件類型？（民事/刑事/行政）
你：民事
MAGI：3. …
      [互動完成後自動產生 Word 檔]

你：翻譯 The court held that the plaintiff failed to establish proximate cause.
MAGI：法院認為，原告未能證明近因關係。
```

### 判決搜尋與研究

```
你：查判決 勞動事件法 調解前置
MAGI：搜尋中…找到 8 筆相關判決：
      [列出判決編號、法院、日期、摘要]

你：深入研究 台灣勞動事件法的舉證責任轉換
MAGI：開始深度研究，預計 2-3 分鐘…
      [完成後回傳多來源研究報告]
```

### 檔案處理

```
[LINE 傳送 PDF 附件]
MAGI：已收到 PDF (3 頁)。
      建議命名：2026.04.05_王大明_民事起訴狀.pdf
      要重命名並歸檔嗎？

你：合併 PDF
MAGI：請傳送要合併的 PDF 檔案（依序傳送，最後說「完成」）

你：OCR 這個圖片
[傳送圖片]
MAGI：辨識結果：
      [輸出辨識的文字內容]
```

## macOS 狀態列

如果你使用 macOS，狀態列（Menu Bar）會顯示 MAGI 系統的即時健康狀態。

### 圖示說明

| 圖示 | 意義 |
|------|------|
| ⚡ MAGI | 全部服務正常 |
| ⚠ MAGI | 部分服務異常 |
| ✗ MAGI | 嚴重異常 |

### 選單內容

點擊狀態列圖示，會展開以下子選單：

- **Core Services** — Daemon、Server、Discord Bot、Tools API 的 PID 及狀態
- **oMLX** — 文字推理（Gemma-4）及 Embedding（BERT）引擎狀態
- **Remote Nodes** — Melchior、Balthasar、Keeper 連線狀態
- **Cron Jobs** — 每個排程任務的最後執行時間（超過 25 小時未執行會標記為過期）
- **NAS** — homes 和 lumi 掛載狀態及磁碟用量
- **Database** — 資料庫容錯狀態（雙活同步 / 本地備份 / 遠端直連）

## 常見問題

### MAGI 沒有回應我的 LINE 訊息

1. 確認 MAGI 服務是否在運行：終端機輸入 `magi status`
2. 檢查 LINE Bot 設定是否正確（webhook URL、channel token）
3. 查看 Discord #magi-logs 頻道是否有錯誤訊息
4. 嘗試重啟：`magi restart`

### MAGI 回覆「處理中」但之後沒有結果

較耗時的任務（深度研究、閱卷、法扶作業）會在背景執行。正常可能需要 1-5 分鐘。如果超過 10 分鐘仍無回覆，可嘗試再次輸入指令。

### MAGI 的回覆看起來不太對

MAGI 使用本地 AI 模型，偶爾可能產生不精確的回覆。特別是：
- **法條引用**：請務必核實 MAGI 引用的法條編號和內容
- **判決摘要**：MAGI 會標記「非權威」的摘要內容，請以原文為準
- **計算結果**：勞基法計算器使用純法定公式，結果可靠

### 狀態列顯示部分服務 DOWN

- **Remote Nodes DOWN**：遠端節點（Melchior/Balthasar/Keeper）離線，不影響本地核心功能
- **NAS NOT MOUNTED**：NAS 未掛載，無法存取案件資料夾。檢查網路連線或 VPN
- **Database 使用備份**：遠端資料庫離線，已自動切換至本地備份。核心功能正常

### 如何更新 MAGI

```bash
magi stop                  # 停止服務
cd ~/Desktop/MAGI_v2       # 進入 MAGI 目錄
git pull origin main       # 拉取最新版本
magi start                 # 重新啟動
```

### 如何回報問題

1. 記錄你輸入的指令和 MAGI 的回覆
2. 查看終端機：`tail -f .agent/server.log`
3. 向系統管理員提供以上資訊

# MAGI 一般使用者手冊

版本：2026-05-18
適用對象：一般使用者、事務協作者、導入顧問
適用版本：MAGI 公開版與私有版

本手冊以「每天實際怎麼用」為主，不要求使用者懂程式。公開版與私有版的基本操作相同；私有版可能額外啟用法扶入口、閱卷、筆錄、NAS、內部通訊頻道與私有法律資料來源。若你的畫面沒有某個功能，通常代表該功能尚未設定、權限不足，或該版本未開啟。

MAGI 是法律與事務作業的 AI 輔助系統，不會取代律師或專業人員的最終判斷。所有正式送出、刪除、批次搬移、公開分享、法律引用與金額計算，都應在送出前由使用者確認。

## 1. MAGI 能幫你做什麼

MAGI 會把案件、檔案、行事曆、待辦、法扶、書狀、摘要、翻譯與法律資料查詢集中在同一個系統中。你可以透過瀏覽器、LINE、Discord、Telegram 或其他已設定的通訊管道使用。

常見用途：

- 查案件資料、開啟案件資料夾、預覽或下載檔案。
- 讀取 PDF、OCR 掃描文件、建議檔名、建立待辦。
- 摘要判決、書狀、合約、研究文獻或會議資料。
- 翻譯文件，並保留法律用語、專有名詞與原文對照。
- 將錄音或影片轉成逐字稿，整理重點與待辦。
- 建立或查詢行事曆事件、OSC 建立待辦與法扶活動統計。
- 管理法扶案件的開辦、進度回報、文件檢查與結案流程。
- 搜尋判決、法條、知識庫、實務見解與指定關鍵字裁判。
- 產生書狀草稿、套用範本，並記錄使用者修正供同案由後續學習。
- 檢查系統健康、磁碟容量、模型狀態、OCR 與資料庫狀態。

MAGI 的回答如果沒有資料來源或工具回傳，就應該說明「查不到」或「資料不足」，不應直接猜答案。

## 2. 開始使用

### 2.1 瀏覽器入口

常見入口如下，實際網址請以你的安裝者提供的設定為準：

```text
http://127.0.0.1:5002/
http://127.0.0.1:5002/health
```

若系統有對外連線設定，可能會提供 Tailscale、Cloudflare 或其他遠端網址。外網打不開時，先確認本機 `/health` 是否正常，再檢查遠端通道。

### 2.2 登入

第一次使用時，管理者會提供登入方式。若系統要求設定密碼，請改成你自己的密碼，不要沿用預設密碼。

若忘記密碼，請找管理者重設。公開版不應內建任何真實帳號或金鑰。

### 2.3 第一次檢查

登入後先看健康狀態：

- 主狀態是否正常。
- 資料庫是否正常。
- 推論服務是否正常。
- OCR 是否正常。
- 檔案根目錄是否可讀寫。
- 行事曆、通知、NAS 或雲端硬碟是否已連上。

若狀態顯示「需檢查」，先處理警示，再做正式作業。

## 3. 主畫面與頁籤

不同版本的名稱可能略有差異，但主要功能通常包含：

- **案件列表**：查看案件卡片、案件狀態、案由、法院案號與資料夾。
- **業務概覽**：查看近期待辦、行事曆事件、OSC 建立待辦與重要提醒。
- **行事曆**：查看庭期、會議、調解、電話、補正期限與自動建立事件。
- **待辦事項**：查看要處理的期限、文件與工作。
- **法扶管理**：查看法扶案件、開辦、進度回報、應備文件與結案。
- **書狀**：查看書狀索引、草擬書狀、範本資料夾與修改紀錄。
- **檔案**：預覽、下載、分享檔案，或查看案件資料夾內容。
- **法律資料**：搜尋判決、法條、知識庫與實務見解。
- **MAGI 狀態**：查看模型、OCR、資料庫、排程與磁碟狀態。

畫面文字應以臺灣慣用語呈現，例如「業務概覽」、「案件列表」、「資料夾」、「對外資料」。若看到不自然或不清楚的用語，請回報給管理者。

## 4. 與 MAGI 對話

### 4.1 自然語言即可

你可以直接說：

```text
請幫我摘要這份 PDF
請把這份英文文件翻成繁體中文
查一下王小明案件最近有什麼待辦
搜尋最高法院關於通譯的判決並分類
幫我產生這件的開庭準備清單
```

MAGI 會判斷應該使用哪個工具。正確行為範例：

- 問行程，要查行事曆或待辦，不應回答天氣。
- 問天氣，才使用網路搜尋或天氣來源。
- 問案件，要查案件資料庫，不應用一般網路搜尋亂找。
- 問判決或法條，要查法律資料工具。
- 查不到時，要說查不到，不應編造。

### 4.2 `@heavy` 或高品質模式

較長、較重要或要交付給外部的人，可以要求高品質處理：

```text
@heavy 請翻譯這份 PDF，保留專有名詞原文，並產生中英對照表
@heavy 請詳細摘要這份監察院報告，列出事實、爭點、結論與可引用段落
```

高品質模式會比較慢，但會做更嚴格的品質檢查。若品質閘門不通過，MAGI 應重新處理或說明失敗原因，不應交付明顯不完整的成品。

### 4.3 需要你補資料時

如果資訊不足，MAGI 可能會問：

```text
請提供案件編號或當事人姓名。
請提供要翻譯的檔案。
同名案件不只一件，請指定法院案號或法扶案號。
```

這不是故障，而是避免把不同案件混在一起。

## 5. 案件管理

### 5.1 查看案件

進入「案件列表」後，可以用以下方式找案件：

- 案件編號。
- 當事人姓名。
- 法院案號。
- 法扶案號。
- 案由或關鍵字。
- 案件狀態。

案件卡片通常會顯示：

- 當事人。
- 案件編號。
- 案由。
- 案件分類。
- 程序或案件種類。
- 法院與法院案號。
- 狀態。
- 法扶標籤。

法扶案件的標籤應清楚顯示，例如「法扶／進行中」、「法扶／已結案」。只用顏色但沒有文字，容易誤解。

### 5.2 進行中、結案準備中與已結案

建議理解如下：

- **進行中**：案件仍在處理，資料夾通常留在進行中區。
- **結案準備中**：已進入結案流程，但還有文件、送出、費用或資料夾搬移要確認。
- **已結案**：案件完成結案，資料夾應移到已結案區，網頁仍能開啟與預覽。

若畫面出現「進行中/結案準備中」這種合併標示，代表分類不夠清楚，應請管理者修正。

案件類別與案由也要分開：案件類別是刑事、民事、行政、非訟、法扶、無償、指定辯護等；「消費者債務清理」、「更生」、「清算」、「傷害」、「貪污治罪條例」這類文字是案由或事件內容，不應被當成案件類別標籤。

### 5.3 同名不同案

同一當事人可能有不同程序或不同案件。MAGI 應以案件編號、法扶案號、法院案號、案由與資料夾位置交叉判斷，不應只靠姓名搬移、結案或統計。

如果 MAGI 顯示「找到多個候選」，請補充更精確資訊。

## 6. 檔案預覽、下載與分享

案件檔案頁應支援：

- 預覽 PDF、圖片、文字檔與常見文件。
- 下載檔案。
- 建立分享連結。
- 複製連結給同事或客戶。
- 開啟檔案所在資料夾。

如果看到：

```text
檔案操作沒有完成
檔案讀取失敗
找不到檔案
```

請先檢查：

1. NAS 或雲端硬碟是否已掛載。
2. 檔案是否剛搬移或同步中。
3. 檔案是否被 Word、PDF 編輯器或其他程式占用。
4. 外網通道是否正常。
5. 健康狀態是否有檔案服務警示。

MAGI 應盡量用穩定的檔案讀取方式處理網路硬碟，避免因檔案鎖定造成預覽、下載或分享失敗。

## 7. PDF、OCR 與檔名

### 7.1 PDF 命名

你可以說：

```text
請幫我命名這份 PDF
請批次整理這個資料夾的法院文件
```

MAGI 會嘗試辨識：

- 文件日期。
- 法院或機關。
- 案號。
- 當事人。
- 文件類型。
- 期限或待辦。
- 是否第一頁是信封或封面。

法院通知、程序裁定、判決、對方書狀常有信封或封面，MAGI 應排除干擾頁，再判斷真正文件內容。

### 7.2 OCR

掃描檔或影像 PDF 需要 OCR。公開版通常使用本機 OCR 或可設定的 OCR 引擎；私有版可能接更完整的 OCR 管線。

OCR 品質不足時，MAGI 應提示需人工複核，不應把亂碼當成正確內容建立待辦或命名。

### 7.3 從 PDF 建立待辦

如果 PDF 檔名或內容包含期限，例如「10 日內補正」、「訂 6 月 1 日下午 4 時調解」，MAGI 可以建立 OSC 待辦或行事曆事件。

若你沒看到新待辦，請確認：

- PDF 是否已進入案件資料夾。
- 自動掃描排程是否有執行。
- PDF 是否命名成功或 OCR 成功。
- 案件是否能被唯一識別。
- Google Calendar 或行事曆整合是否正常。

## 8. 摘要、翻譯與逐字稿

### 8.1 摘要

你可以要求：

```text
摘要這份 PDF
整理成三點
請列出事實、爭點、法院理由與結論
請做可以貼給客戶看的白話摘要
```

好的摘要應包含：

- 文件類型。
- 主要事實。
- 爭點。
- 理由或依據。
- 結論。
- 期限或下一步。
- 可引用段落或頁碼。

若摘要只有空泛開場、語言錯誤、把文件內容講錯，應重跑或回報。

### 8.2 翻譯

你可以要求：

```text
請翻譯成繁體中文
請產出中英對照表
請保留每個專有名詞的原文
請用法律文件語氣翻譯
```

MAGI 的高品質翻譯應做到：

- 段落順序不亂。
- 不漏段。
- 專有名詞後保留原文，例如「成癮者（addicts）」。
- 法律用語符合臺灣慣用語。
- 不把普通語句誤譯成奇怪語意。
- 產出檔案內保留術語表或專有名詞表。

如果你看到錯譯，例如語意明顯不通，請直接指出「這句應改成……」。MAGI 應記錄修正，供同類型翻譯品質改善。

### 8.3 逐字稿

你可以上傳音訊或影片，要求：

```text
請轉逐字稿
請整理會議紀錄和待辦
請把逐字稿整理成摘要
```

好的逐字稿應包含：

- 可讀的段落。
- 說話者或時間資訊，若來源可辨識。
- 不確定詞標示。
- 待辦與決議整理。

音訊品質太差時，MAGI 應提示需人工複核。

## 9. 書狀、範本與修正學習

### 9.1 書狀草擬

你可以說：

```text
幫我草擬起訴狀
請依這件案件資料草擬陳報狀
請用範本產生聲請狀
```

MAGI 應先讀取案件資料、已歸檔文件與書狀範本，再產生草稿。草稿應保留可修改空間，不應直接當成完稿送出。

### 9.2 範本資料夾

範本是特殊資料，不是一般案件。範本頁籤應能：

- 查看範本清單。
- 預覽範本。
- 下載範本。
- 分享範本連結。
- 複製給他人。

範本不應顯示一般案件狀態，也不應把測試用分類當成真實案由。

### 9.3 修正學習

如果你修改了 MAGI 產出的書狀，可將修改前後差異回饋給 MAGI。學習規則：

- 只套用在同案由、同文件類型。
- 不同案由不可混用。
- 不應覆蓋使用者明確指定的內容。
- 學習紀錄應可追溯、可清除。

## 10. 行事曆與待辦

業務概覽應分開顯示：

- **OSC 建立待辦**：由案件文件、期限、PDF 命名或系統規則建立。
- **行事曆事件**：由 Google Calendar、Apple Calendar 或外部行事曆匯入。

兩者不能混在同一個框裡，否則使用者會分不清是文件期限還是行程。

常見操作：

```text
今天有什麼事
下週有哪些庭
查某案件的待辦
把這件標示已完成
```

行事曆同步應避免把同事手動登錄的私人行程全部混入案件待辦。一般案件事件應有案件編號；法扶活動計數事件則需由 DB 確認是法扶案件。

## 11. 法扶作業

公開版可提供法扶流程框架；私有版若已設定帳號與權限，可能可直接處理實際案件。

常見功能：

- 新法扶派案建立。
- 官網附件下載。
- 開辦。
- 應備事項表。
- 消債文件確認。
- 進度回報。
- 結案。
- 已結案資料夾搬移。

重要原則：

- 開辦是正式流程，不應出現「只暫存」的舊流程混淆。
- 已結案案件搬移後，網頁仍應能開啟資料夾與檔案。
- 同名不同案件不可誤搬。
- 消債所得清單應依年度自動更新，例如民國 115 年 5 月後可列 113、114 年；下一年類推。
- 進度回報提醒若已回報，應可用「某案已回報」冷卻一段期間後再提醒。

## 12. 閱卷與筆錄

私有版若已設定司法院登入與下載流程，可以處理：

- 閱卷申請。
- 閱卷通知檢查。
- 可下載檔案下載。
- 到院閱卷通知整理。
- 筆錄下載與更名。
- 歸檔到案件資料夾。

閱卷去重原則：

- 已下載且已歸檔的案件，不應一直重複通知。
- 只有繳費單的日期資料夾，不應算成實際閱卷次數。
- 同案多列通知應合併呈現。

筆錄通知原則：

- 只列出有下載到檔案的案件。
- 不要列一長串 0 份案件造成閱讀負擔。

## 13. 法律資料、判決搜尋與通譯實證研究

### 13.1 判決與法條

你可以說：

```text
查民法第184條
搜尋最高法院關於背信罪的判決
找消費者債務清理的實務見解
```

MAGI 應使用法律資料工具或已設定的法律資料來源。查不到時應說查不到，不應用模型記憶補編。

### 13.2 通譯實證研究

你可以說：

```text
請用關鍵字「最高法院 通譯」上網抓取裁判並分類
請把最高法院提到通譯的判決整理成表格
請區分只是條文帶過、通譯品質爭議、未用通譯、外語證據翻譯與法院結果
```

MAGI 會：

1. 到司法院裁判系統搜尋指定關鍵字。
2. 抓取裁判全文。
3. 整理成 TXT。
4. 依通譯相關原文分類。
5. 輸出 CSV、XLSX、Markdown。
6. 保留來源 URL、裁判字號、法院結果與原文摘錄。

若你指定不同關鍵字，MAGI 應依你的命令抓取與分類，不應只讀本機既有資料夾。

## 14. 帳務與支出

## 18. 常見問題

### Q1：MAGI 查不到資料，可以叫它猜嗎？

不建議。法律與案件資料應以可驗證來源為準。MAGI 查不到時應回覆查不到，或請你補資料。

### Q2：檔案預覽或下載失敗怎麼辦？

先確認檔案是否存在、NAS 是否掛載、外網是否正常、檔案是否被其他程式占用。若 `/health` 顯示檔案服務異常，請先處理健康狀態。

### Q3：翻譯品質不好怎麼辦？

使用 `@heavy` 或明確要求「保留專有名詞原文」、「中英對照表」、「法律文件語氣」。若仍錯誤，指出錯句與正確譯法，讓 MAGI 記錄修正。

### Q4：行事曆事件和 OSC 待辦有什麼差別？

行事曆事件是時間行程，例如開庭、會議、電話。OSC 待辦是系統依案件或文件建立的工作，例如補正、上訴期限、資料待補。兩者應分框顯示。

### Q5：為什麼同名案件要我再確認？

因為同一人可能有多個程序或多個案件。MAGI 要避免把 A 案的結案、待辦或法扶統計誤套到 B 案。

### Q6：可以把 MAGI 對外給客戶用嗎？

可以，但應先完成公開版隔離檢查、安裝精靈、健康檢查、煙霧測試、資料脫敏、金鑰清查與操作手冊確認。不要把私有資料夾或 `.env` 交給客戶。

## 21. AI 對話與工具調用詳解

### 21.1 一般問答

用途：回答一般問題、整理短文字、協助判斷下一步應該使用哪個工具。

入口：MAGI 網頁聊天、LINE、Discord、Telegram 或已設定的通訊軟體。

操作方式：直接輸入完整問題。若問題需要資料來源，請提供案件編號、檔案、網址或關鍵字。

範例：

```text
這份裁定的重點是什麼？
我明天有沒有開庭？
請查這件案件的待辦。
```

輸出：MAGI 會回覆答案，並在需要時列出資料來源或使用過的工具。

注意事項：

- 法律結論、案件事實、法條與判決引用必須有資料來源。
- 工具沒有回傳資料時，MAGI 應說明資料不足，不應補編。
- 問題太模糊時，MAGI 應要求補資料。

### 21.2 工具調用

用途：讓 MAGI 用正確工具處理不同任務，例如查行事曆、查案件、搜尋判決、翻譯、摘要、讀檔。

常見工具對應：

| 你問的內容 | 應使用的工具 |
| --- | --- |
| 行程、開庭、會議 | 行事曆或待辦工具 |
| 天氣、最新資訊、股價 | 網路搜尋或市場工具 |
| 案件狀態、當事人 | 案件資料庫 |
| 判決、裁判、實務見解 | 判決搜尋或法律資料工具 |
| 法條 | 法條搜尋 |
| 摘要 | 摘要工具 |
| 翻譯 | 翻譯工具 |
| 本機檔案 | 檔案讀取工具 |
| 通譯判決實證研究 | 通譯實證分類工具 |

注意事項：

- 問行程時，不應查天氣。
- 問案件時，不應上網亂搜。
- 查不到時應回覆查不到。
- 高風險操作不得自動送出或刪除，應等待確認。

### 21.3 高品質模式 `@heavy`

用途：處理正式文件、長文翻譯、重要摘要、研究文獻、法律文件與需要較高品質的輸出。

操作方式：在訊息前加上 `@heavy` 或明確要求高品質處理。

範例：

```text
@heavy 請翻譯這份英文 PDF，產出中英對照表並保留專有名詞原文。
@heavy 請摘要這份監察院報告，列出事實、爭點、結論與可引用段落。
```

輸出：通常會產出較完整的文字、表格或文件檔，並經過品質閘門檢查。

注意事項：

- 高品質模式較慢。
- 若品質檢查不通過，MAGI 應重新處理或說明原因。
- 使用者仍應審閱正式交付文件。

## 27. 摘要、翻譯、逐字稿與文件產出功能


- 長文應保留章節脈絡。
- 法律文件應列出主文、理由、程序與期限。

用途：翻譯法律文件、研究文章、合約、信件與一般文字。

範例：

```text

翻譯這段英文，保留每個專有名詞原文。
```

輸出：翻譯文字、對照表、術語表或 DOCX。

品質要求：

- 專有名詞後應保留原文。
- 不漏段、不亂段。
- 法律用語採臺灣慣用語。
- 翻譯錯誤應可被使用者回饋修正。

### 27.3 逐字稿


輸出：逐字稿、摘要、待辦、時間資訊。

品質要求：

- 音訊太差應標示不確定。
- 不應輸出空白或極短內容當作完成。
- 重要會議應保留原始檔與轉錄檔。

### 27.4 DOCX / Word 文件

### 27.5 Excel / 表格

### 27.6 簡報

```text
搜尋侵權行為損害賠償舉證責任的判決。
收集消費者債務清理的實務見解。
```

輸出：判決清單、摘要、來源、入庫狀態。

注意事項：

- 程序性文書、抓不到見解、提示詞洩漏內容應被排除或降噪。
- 大量收集應避開工作時間，避免造成負載。

### 29.2 司法院網頁搜尋

用途：直接到司法院裁判系統搜尋關鍵字，抓取裁判清單與全文。

```text
司法院搜尋 最高法院 通譯。
抓取這筆裁判全文。
```

輸出：裁判標題、網址、全文 TXT。

- 司法院網站可能限流或維護。
- 舊裁判可能因網站改版需要備援來源。

### 29.3 流程型裁判搜尋歸檔

用途：把自然語句轉成司法院查詢，抓清單與全文，建立 manifest 與 TXT 歸檔。

範例：

```text
請用「最高法院 通譯」抓取裁判並歸檔。
```

輸出：歸檔資料夾、manifest、TXT、摘要。

注意事項：

- 大量抓取要設定上限與間隔。

### 29.4 法條搜尋

用途：搜尋法規條文與相關主題。

範例：

```text
查民法第184條。
搜尋消費者債務清理條例關於所得清單。
```

輸出：條文內容、相關條文、來源。

```text
幫我試算裁判費。
```

注意事項：

- 計算結果仍需依最新規定與個案確認。

### 29.6 證據能力分類

```text
請把這份卷證索引做證據能力分類。
```

注意事項：

- 證據能力意見需律師確認。
- OCR 錯誤會影響分類。

- 通譯角色。

- 來源檔案與 PDF。

注意事項：

- 指定關鍵字時，MAGI 應上網抓取，不應只讀桌面既有資料。

## 30. 法扶功能詳解

用途：提醒進行中逾一定期間的法扶案件回報進度。

```text
1130619-T-027 已回報
```

### 30.6 法扶結案

用途：檢查結案依據、上傳附件、完成報結與資料夾搬移。

輸出：結案狀態、缺檔清單、上傳結果、資料夾搬移結果。

注意事項：

- 已結案資料夾仍要能在網頁開啟。

### 30.7 法扶撤回、延長、費用、條件回報

用途：處理法扶非結案類回報流程。

輸出：預覽資料、附件檢查、只存草稿或正式送出狀態。

用途：把不同類型訊息送到正確頻道。

分類：

- 系統健康：系統通知。
- 法扶派案：法扶派案。
- 法扶進度回報：進度回報。
- 閱卷與筆錄：業務通知。
- 股市與情報：資訊頻道。
- 嚴重故障：管理員警報。

注意事項：

- 系統巡檢不應送到業務頻道。
- 測試訊息應標明測試或不要推送。

### 33.2 LINE / Discord / Telegram

用途：透過通訊軟體下指令、接收附件、推送通知。

常見操作：

```text
幫我摘要這份 PDF。
系統狀態。
某案已回報。
```

注意事項：

- LINE 有回應時間限制，長任務會背景處理。
- Discord 適合長文字與表格。
- Telegram 常用於推播與狀態通知。

### 33.3 對外資料

用途：產生可複製、可分享給客戶或外部對象的文字與檔案連結。

輸出：對外摘要、附件連結、注意事項。

注意事項：

- 對外資料應脫敏或確認可分享。
- 分享前確認連結權限。

## 34. 系統健康、維運與安全功能

### 34.1 健康檢查

用途：查看 MAGI 是否可正常運作。

檢查項目：

- 主狀態。
- 資料庫。
- 推論服務。
- OCR。
- 向量資料庫。
- NAS 或檔案掛載。
- 日常稽核。
- 維運健康。

操作：

```text
系統狀態
健康檢查
```

注意事項：

- 黃燈可使用但需處理。
- 紅燈應停止正式送出與批次搬檔。

### 34.2 偵測精靈

用途：檢查安裝環境、依賴、資料庫、模型、OCR、檔案路徑與公開版隔離。

範例：

```bash
python3 scripts/magi_doctor.py --json
```

輸出：檢查報告與建議修復項目。

### 34.3 程序衛生

用途：偵測殭屍程序、重複 daemon、port 佔用、卡死程序。

範例：

```text
檢查是否有殭屍程序。
```

輸出：程序清單、可修復項目、修復結果。

注意事項：

- 強制關閉程式前需確認是否會造成未儲存文件遺失。

### 34.4 自動巡檢與夜間任務

用途：夜間整理 PDF、法扶、閱卷、筆錄、判決、快取與健康狀態。

輸出：夜間健康報告、失敗項目、待人工處理清單。

注意事項：

- 報告應有意義，不只列數字。
- 持續失敗應進 issue agenda。

### 34.5 磁碟清理

用途：清理快取、臨時輸出、重複下載檔、舊報告與可重建資料。

可清理：

- 瀏覽器快取。
- OMLX / 模型暫存。
- 過期 exports。
- 去重後多餘繳費單複本。
- 可重建的 OCR 中間檔。

不可清理：

- 案件原始資料。
- 使用者資料。
- DB。
- Paperclip 單機版必要 JSON。
- 未確認用途的備份。

### 34.6 NAS 掛載

用途：確認案件資料所在網路硬碟可用。

檢查項目：

- 掛載名稱正確。
- 不要掛成錯誤別名。
- 讀寫正常。
- 負載沒有異常暴增。

注意事項：

- NAS 未掛載時，檔案預覽、下載、分享、歸檔都可能失敗。
- 自動重開機後應自動掛回。

### 34.7 模型日夜切換

用途：日間使用輕量快速模型，夜間使用較大模型處理批次任務。

注意事項：

- 日間主模型應符合設定，不應無故跑夜間大模型。
- 切換前後應檢查模型服務健康。
- 若使用者正在工作，可暫停本次重開，但其他排程仍應保留。

### 34.8 Iron Dome 與安全掃描

用途：阻擋危險操作、敏感資訊外洩、公開版混入私有資料。

操作：

```text
執行公開版隔離檢查。
檢查是否有金鑰外洩。
```

輸出：通過、警告或阻斷原因。

注意事項：

- 公版推送前必跑。
- 私有 `.env`、token、cookie、案件資料不得進 git。

### 34.9 技能建立與技能內化

用途：管理員可讓 MAGI 建立新技能、匯入外部工具或把常用知識整理成技能。

注意事項：

- 一般使用者不建議操作。
- 新技能需經安全掃描與測試。
- 不可自動安裝不可信來源。

### 34.10 自我修復與核心改動審批

用途：MAGI 夜間可提出修復或改善建議，等待管理者批准。

範例：

```text
批准 ccr-xxxxxxxx
拒絕 ccr-xxxxxxxx 原因：需要人工確認資料完整性
```

注意事項：

- 資料庫去重、重新索引、批次清理等核心改動應先 dry-run。
- 要有回滾、備份與驗證。

## 35. 公開版與私有版差異

### 35.1 公開版

公開版適合外部使用者自行安裝與試用。它應包含：

- 安裝精靈。
- 偵測精靈。
- 一般案件、檔案、PDF、摘要、翻譯、法律資料與健康檢查框架。
- 範例設定。
- 操作手冊。
- 隱私權政策、服務條款、資料保留政策與第三方套件清單。
- 公版隔離稽核、乾淨公開版安裝檢查與商用 smoke gate。

公開版不應包含：

- 私有案件資料。
- 私有 NAS 路徑。
- 私人帳號。
- 金鑰、token、cookie。
- 私有實務見解來源。
- 私有法扶登入資訊。

### 35.2 私有版

私有版可額外啟用：

- 真實案件資料庫。
- NAS 與雲端同步。
- 法扶入口。
- 閱卷與筆錄登入。
- 私有通訊頻道。
- 私有法律資料來源。
- 自動報表與內部排程。

注意事項：

- 私有版改動推公開版前必須脫敏。
- 公版與私版 commit 應分開管理。
- 私有版通過商用檢查後，仍應確認公開版不含私有資料、金鑰、NAS 路徑或私有實務見解來源。

### 37.7 錯誤處理範例

| 看到的狀況 | 建議說法或處理 |
| --- | --- |
| 檔案預覽失敗 | `檢查這份檔案為何無法預覽，並確認 NAS 是否掛載` |
| 分享連結打不開 | `檢查分享服務與外網通道` |
| 行事曆沒新增 | `檢查這件 PDF 是否有建立 OSC 待辦與 Google 行事曆事件` |
| 法扶案號不符 | `重新比對法扶案號、案件編號、當事人與資料夾` |
| 翻譯品質不佳 | `請用 @heavy 重跑，保留專有名詞原文並輸出中英對照表` |
| 摘要過短或漏段 | `請依原文段落重新摘要，列出頁碼與可引用段落` |
| 系統黃燈 | `檢查 MAGI 健康狀態並列出需要處理的項目` |
| 公開版稽核失敗 | `執行 public_release_audit 並逐項修正 errors/warnings` |

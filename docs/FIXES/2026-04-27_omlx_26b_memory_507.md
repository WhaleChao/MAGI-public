# oMLX 26B Completion 507 修復

## 根因

驗證 Chandra/pdf-namer live smoke 時，`pdf-namer` 呼叫 text oMLX 8080 出現 507。直接打 `/v1/chat/completions` 後確認錯誤為：

`Model 'gemma-4-26b-a4b-it-4bit' (14.63GB) exceeds max-model-memory (14.00GB)`

`/v1/models` 與 MAGI `/health` 仍顯示 reachable，所以這是「健康綠但實際 completion 紅」的夜間 26B LaunchAgent 設定問題。

## 修復

- `~/Library/LaunchAgents/com.magi.omlx.plist`
  - `OMLX_TEXT_MAX_MODEL_MEMORY`: `14GB` → `16GB`
  - `OMLX_TEXT_MAX_PROCESS_MEMORY`: `12GB` → `18GB`
- `~/Library/Application Support/MAGI/bin/omlx_switch_model.sh`
  - night mode 同步改為 16GB / 18GB。
  - Layer 3 RSS check 上限同步調整為 16GB。
- `config/bin/omlx_switch_model.sh`
  - repo 版 night mode 同步改為 16GB / 18GB。
- `CLAUDE.md`
  - 新增 standing 規則：26B 夜間上限不可低於模型實際大小。

## 驗證

- `bash -n config/bin/omlx_switch_model.sh`
- `bash -n ~/Library/Application Support/MAGI/bin/omlx_switch_model.sh`
- `launchctl bootout/bootstrap/kickstart com.magi.omlx`
- `launchctl print` 確認 8080 PID fresh 且環境為 `16GB/18GB`
- `/v1/models` 成功列出 `gemma-4-26b-a4b-it-4bit`
- `/v1/chat/completions` 不再回 507
- `pdf-namer --task self_test` 不再出現 507，結果 success true
- `/health` 維持 `status=operational`

# brief-gen

書狀輔助產生技能，依案件類型與爭點自動產生書狀草稿架構。

## 指令

| 指令 | 說明 |
|------|------|
| `--task draft --text "案件描述"` | 產生書狀草稿架構 |
| `--task template --text "書狀類型"` | 列出可用書狀範本 |
| `--task enrich --text "案號"` | 從案件資料自動充實書狀內容 |
| `--task export --text "案號" --mode docx` | 匯出為 Word 文件 |

## 書狀類型

| 類型代碼 | 說明 |
|----------|------|
| `complaint` | 起訴狀 / 民事起訴狀 |
| `answer` | 答辯狀 |
| `appeal` | 上訴狀 / 上訴理由狀 |
| `motion` | 聲請狀（假扣押、假處分、保全證據等） |
| `closing` | 辯論意旨狀 / 最後言詞辯論狀 |
| `statement` | 準備書狀 / 陳報狀 |
| `labor` | 勞動事件相關書狀（調解聲請、勞資爭議） |

## 流程

### draft — 產生草稿
1. 解析案件描述（案由、當事人、爭點）
2. 自動判斷適用書狀類型
3. 查詢相關法條（statutes-vdb）
4. 查詢相關判決見解（judgment-collector）
5. 產生結構化草稿：
   - 案號、當事人、案由
   - 訴之聲明 / 答辯聲明
   - 事實及理由（分項列點）
   - 適用法條引用
   - 證據清單
   - 附件清單

### enrich — 充實內容
1. 從案號定位案件資料夾
2. 讀取既有書狀、筆錄
3. 提取歷次爭點與法院心證
4. 自動補充事實及理由段落

### export — 匯出
1. 將草稿轉為 docx 格式
2. 套用台灣法院書狀格式（字型、行距、邊界）
3. 儲存至案件資料夾

## 書狀格式規範
- 字型：標楷體 14pt
- 行距：1.5 倍
- 邊界：上下左右各 2.5cm
- 頁首：案號 + 書狀類型
- 附件編號：依序「附件一、附件二...」

## 依賴
- `statutes-vdb` skill（法條查詢）
- `judgment-collector` skill（見解查詢）
- `transcript-indexer` skill（筆錄搜尋）
- `docx` skill（Word 匯出）
- `law_review` skill（法律用語審核）

## LINE/DC 指令格式
- 「寫書狀」「草稿」「擬狀」→ draft
- 「書狀範本」「範本清單」→ template
- 「匯出書狀」→ export

## 呼叫格式
觸發詞：書狀、撰狀、草稿、起訴狀、答辯狀
參數：case=案號, type=書狀類型(選填), requirements=要求(選填)

## 呼叫範例
使用者：幫 2025-0087 寫民事準備書狀
→ 撰狀 case=2025-0087 type=民事準備書狀

使用者：2025-0004 的刑事聲明抗告狀
→ 撰狀 case=2025-0004 type=刑事聲明抗告狀

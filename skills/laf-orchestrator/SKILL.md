## 指令

```bash

python action.py --task closing --laf-case-no "1140806-J-002" --client "陳賜聰"

# 帶理由（當有零次數欄位時需要）
python action.py --task closing --laf-case-no "1141029-E-015" --client "莊依稜" --reason "偵查中案件依法不得閱卷"

# 開辦回報

python action.py --task condition --laf-case-no "..." --client "..."

# 查看次數（不執行 portal 操作）
```

## 結案報結流程（Closing Report）規則

### 1. 次數統計來源

| 項目 | 主要來源 | 補充來源 |
|------|----------|----------|
| 開會/律見 | `meetings` 表 + `case_todos`（律見） | `calendar_events`（開會/律見關鍵字） |
| 電話聯繫 | `case_todos`（聯繫/通話） | `calendar_events`（電話/聯繫關鍵字） |
| 開庭 | `case_todos`（言詞辯論/準備程序/...） | `calendar_events`（開庭相關關鍵字） |
| 閱卷 | `case_todos`（閱卷） | `06_閱卷資料` 子資料夾日期、`calendar_events` |
| 書狀 | `04_我方歷次書狀` 資料夾檔案數 | — |

### 2. 日曆事件分類關鍵字

```
開庭: 開庭, 言詞辯論, 準備程序, 審理程序, 調解, 訊問, 協商程序, 調查
開會/律見: 律見, 會議, 開會, 會面, 接見
電話聯繫: 電話, 聯繫, 通話, 聯絡
閱卷: 閱卷
```

### 3. 排除規則

| 規則 | 說明 |
|------|------|
| **開辦日過濾** | 只計算 `cases.start_date`（開辦日期）之後的活動 |
| **宣判不計入開庭** | 日曆摘要含「宣判」或「宣示判決」不計入開庭次數 |
| **颱風假/放假日排除** | `calendar_events` 中 `is_all_day=1` 且摘要含「颱風/停班/放假/停課/天然災害」的日期，開庭不計入 |
| **聲請改期排除** | 日曆摘要含「聲請改期/取消/改期/不到庭/不出席」不計入 |
| **法扶行政事件排除** | 日曆摘要含「法扶開辦末日/上訴/再議」不計入 |
| **去重** | 同日同摘要的日曆事件只計一次 |
| **開會與律見合併** | 開會次數與律見次數合併為 `meet_times`，法扶表單的「律見次數」(`inq_times`) 設為 0 避免重複計算 |

### 4. 偵查中案件特殊規則

- **閱卷次數一定是 0**：臺灣法規不准偵查中閱卷，這是法律事實
- `#noarrivereason`（扶助律師特別說明）自動填入「偵查中案件依法不得閱卷。」
- 偵查案件判斷：`case_reason` 含「偵查」「偵辦」「毒品」關鍵字
- **不需要律師確認閱卷=0 的理由**

### 5. 零次數處理流程

法扶表單 `checkData()` 在以下欄位為 0 且 `#noarrivereason` 為空時會彈 modal 阻擋暫存：
- `disc_times`（研討案情 = 面談 + 電話）
- `viewsheet_times`（閱卷）
- `ap_times`（開庭）
- `wc_times`（書狀）

**處理流程**：
1. 系統偵測到零次數欄位 → 通知律師確認
2. 律師回覆說明文字 → 系統填入 `#noarrivereason`
3. 偵查中閱卷=0 例外：自動填入，不需律師確認
4. 若 modal 仍出現 → 關閉 modal 後直接以 `tempSaveFlag='Y'` 提交（等同暫存）

### 6. 表單存檔策略

| 操作 | JS 函式 | 說明 |
|------|---------|------|
| 第一頁存檔 | `checkData('0')` | 報結類型/法院/案號/日期等 |
| 第二頁暫存 | `doTempSave()` → `checkData()` → `form.submit()` | 次數/日期 |
| 第二頁送出 | `doFinalSave('N')` | **禁止自動執行** |

> **安全策略**：CASPER 一律只執行暫存（`doTempSave`），不執行送出（`doFinalSave`）。律師需自行上官網確認後手動送出。

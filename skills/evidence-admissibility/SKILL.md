# 卷證索引證據能力分類技能

本技能將法院刑事案件卷證索引（通常為審判筆錄中法官逐一提示證據的文字紀錄）自動解析，並依台灣刑事訴訟法傳聞法則，為每筆證據標註被告方的證據能力意見，最終產出格式化的 Excel 檔案。

## 適用場景

辯護律師收到法院卷證索引後，需要逐一對每筆證據表示證據能力意見。本技能自動化此流程：

1. 解析卷證索引文字檔（法院審判筆錄格式）
2. 辨識證據所屬人物/組別、證據類型、卷頁出處
3. 依傳聞法則規則自動分類證據能力意見
4. 產出含色碼標記的 Excel 表格

## 工作流程

### 第零步：定位案件（自動 or 手動）

**模式 A — 自動查詢（優先）：**

當使用者提到案號或當事人名時，先執行 `lookup` 指令：
1. 用案號/當事人名查詢 OSC 資料庫（`cases` 表）
2. 取得 `folder_path` → 透過 `case_path_mapper` 轉為本機 NAS 路徑
3. 在案件資料夾中掃描含「卷證索引」「證據索引」「證據清單」等關鍵字的 .txt 檔
4. 列出找到的檔案供使用者確認

範例觸發語句：
- 「幫我做王大明案的證據能力」→ lookup {"client_name":"王大明"}
- 「2025-0088 的卷證索引證據能力」→ lookup {"case_number":"2025-0088"}

**模式 B — 手動提供：**

使用者直接提供檔案路徑或貼上卷證索引文字內容，跳過查詢步驟。

### 第一步：收集必要資料

向使用者確認以下資料（缺一不可）：

1. **卷證索引文字檔**（.txt）— 法院審判筆錄中「審判長問」逐一提示證據的完整文字紀錄（第零步可自動定位）
2. **被告姓名** — 本案中代表哪位被告（影響「被告本人」vs「被告以外之人」的判斷）
3. **檢察官補充理由書**（PDF）— 用於辨識檢察官援引了哪些偵訊筆錄作為證據方法（通常在證據清單的「壹、供述證據」部分）
4. **已於審判中作證的證人清單** — 哪些證人已經法院傳喚到庭作證（有審判筆錄），代表已合法調查
5. **秘密證人名單**（如有）— 哪些證人為秘密證人，其陳述非親自見聞

如果使用者無法提供第3項，可以請使用者手動提供「檢察官援引的證人/被告名單」。

**備註：** 若第零步已定位案件資料夾，也可在資料夾中掃描補充理由書 PDF。

### 第二步：解析卷證索引

讀取 `scripts/build_evidence_xlsx.py` 腳本，理解其解析邏輯，然後依使用者提供的資料進行客製化：

**解析核心邏輯：**

- 以 `審判長問` 作為切割點，將全文分成多個段落（sections）
- 每個段落對應一位證人/被告的全部證據，或一組書證/物證
- 段落內以中文數字序號（`一、`、`二、`...）切割個別證據項目
- 每筆證據提取：證據名稱、證據類型（筆錄類型）、卷頁出處（括號內容）

**段落辨識規則需要客製化的部分：**
- `section_rules` 列表：依卷證索引中出現的人物順序建立，包含搜尋字串、段落名稱、段落類型（witness/codefendant/defendant_self/documentary/witness_secret）、人物ID
- `PROSECUTOR_ADOPTED_PERSONS`：從補充理由書中提取檢察官援引的人物名單
- `PERSONS_WITH_TRIAL_TRANSCRIPT`：使用者提供的已於審判中作證的證人名單

### 第三步：分類證據能力

詳細規則請讀取 `references/admissibility_rules.md`。以下為摘要：

| 優先序 | 條件 | 意見 |
|--------|------|------|
| 1 | 被告本人的任何筆錄 | 不爭執證據能力 |
| 2 | 審判筆錄、言詞辯論筆錄 | 不爭執證據能力 |
| 3 | 準備程序筆錄、協商程序筆錄 | 不爭執證據能力 |
| 4 | 訊問筆錄（法院階段：國審強處/訴字） | 不爭執證據能力 |
| 4' | 訊問筆錄（偵查階段：偵字/相字/他字） | 繼續適用下方傳聞法則 |
| 5 | 書證/物證 | 不爭執證據能力 |
| 6 | 被告以外之人的警詢筆錄 | 傳聞，無證據能力（§159 I） |
| 7 | 秘密證人的偵訊筆錄 | 非親自見聞，無證據能力（§159-1 II） |
| 8a | 偵訊筆錄—已合法調查（有審判筆錄） | 傳聞，不同意（§159-1 II） |
| 8b | 偵訊筆錄—檢察官援引但未合法調查 | 未合法調查，不得作為判斷依據（§155 II） |
| 8c | 偵訊筆錄—檢察官未援引 | 傳聞，無證據能力（§159 I、§159-1 II） |

**偵查階段判斷：** 卷宗字號含「偵字」、「相字」、「他字」者屬偵查階段；含「國審強處」、「訴字」者屬法院階段。

### 第四步：產出 Excel

使用 openpyxl 產出 Excel 檔案，包含以下欄位：

| 欄位 | 說明 |
|------|------|
| 編號 | 流水號 |
| 證據出處（人/組別） | 該證據所屬的證人/被告名稱或書證/物證 |
| 證據名稱 | 證據的具體描述（已清理程序性文字） |
| 證據類型 | 警詢筆錄/偵訊筆錄/審判筆錄/訊問筆錄/書證 等 |
| 卷頁出處 | 括號內的卷冊及頁碼資訊 |
| 與被告○○○之關係 | 被告本人/被告以外之人—同案被告○○○/被告以外之人—證人○○○ |
| 證據能力意見 | 依上述規則產出的意見 |

**格式化規則：**
- 標題列：深藍底白字，置中，凍結窗格
- 無證據能力項目：淺黃色底（hearsay_fill = FFF2CC）
- 秘密證人項目：淺粉色底（secret_fill = FCE4EC）
- 一般項目：白色/淺藍交替
- 啟用自動篩選

### 第五步：文字清理

解析時需清理以下程序性文字（這些不是證據內容）：

- `對於下列證據，有何意見？`
- `（提示並告以要旨）` / `提示並告以要旨`
- `是否實在？是否出於自由意志？有無意見？`
- 供述程序文字（如 `在警詢、偵查中及本院庭期中之供述`）
- 所有斷行符號（`\n`）及多餘空白
- 中文數字序號前綴（`二、`）不寫入表格

卷頁出處欄位中也需過濾 `提示並告以要旨`。

## 客製化指引

每個案件的以下內容都不同，需要根據使用者提供的資料調整：

1. **section_rules**：根據卷證索引中出現的人物和順序重新建立
2. **PROSECUTOR_ADOPTED_PERSONS**：從該案的補充理由書提取
3. **PERSONS_WITH_TRIAL_TRANSCRIPT**：由使用者確認
4. **秘密證人名單**：由使用者確認
5. **被告姓名**：影響 defendant_self 的判斷和表格標題

## 腳本位置

核心解析腳本位於 `scripts/build_evidence_xlsx.py`，可作為模板使用。執行前請先根據本案資料客製化上述變數，再執行腳本產出 Excel。

## 法條引用格式

所有法條引用必須使用正式全稱格式：
- 正確：`刑事訴訟法第159條第1項`
- 錯誤：`刑訴§159`、`刑訴§159I`

最高法院判決引用格式：`最高法院114年度台上字第3941號判決`

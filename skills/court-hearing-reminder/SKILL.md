# court-hearing-reminder

開庭提醒、補正期限、繳費期限 — 統一排程提醒與庭前準備摘要自動生成。

## 能力
- 每日掃描 case_todos 中的排程（開庭/補正/繳費）
- **繳費/補正**：從入庫第一天起每日推播，直到截止日或標記完成
  - 倒數 ≥4 天 → info
  - 倒數 3 天以內 → warning
  - 當天截止 → critical
- **開庭**：前一天 + 當天提醒
- 附帶案件摘要：當事人、案由、法院、歷次判決見解（品質把關，寧缺勿濫）
- 支援指定案件標記完成（關閉警報），同時期多筆繳費可各別關閉

## 任務
- `--task scan`：掃描未來 7 天排程，回傳 JSON
- `--task remind`：發送提醒通知
- `--task prep --case-number XXX`：生成庭前準備摘要（支援案號/當事人/法院案號搜尋）
- `--task list`：列出所有待辦排程
- `--task done --text "張國賢繳了"`：標記特定案件已完成，關閉警報

## 支援的 todo_type
- `開庭`：法院開庭排程
- `補正`：法院要求補正文件的期限
- `繳費`：裁判費、執行費等繳費期限

## 通訊軟體觸發
- 「最近有什麼庭」「開庭排程」「庭期」→ list
- 「準備 XXX 案的開庭資料」→ prep
- 「補正期限」「繳費提醒」→ list
- 「張國賢繳了」「補字第54號交了」「關掉XX提醒」→ done（精確匹配單筆，多筆時列出選擇）

## 呼叫格式
觸發詞：開庭、開庭提醒、庭期
參數：action=動作(check/remind), date=日期(選填)

## 呼叫範例
使用者：今天有開庭嗎
→ 庭期 action=check date=today

使用者：這週的開庭行程
→ 庭期 action=check date=this_week
